Package Manager

Der Dynamo Package Manager ist eine zentrale Anlaufstelle für das Durchsuchen, Installieren und Publizieren von Paketen. Sie können über die obere Navigationsleiste unter Pakete > Package Manager darauf zugreifen.

Was ist ein Paket?

Dynamo bietet eine große Anzahl sofort einsatzbereiter Funktionen und verfügt außerdem über eine umfangreiche Paketbibliothek, mit der sich die Funktionen von Dynamo erheblich erweitern lassen. Ein Paket ist eine Sammlung benutzerdefinierter Blöcke oder zusätzlicher Funktionen. Der Dynamo Package Manager ist ein Portal für die Community zum Herunterladen aller online publizierten Pakete. Diese Toolsets werden sowohl von Drittanbietern als auch vom Dynamo-Team entwickelt, um die Kernfunktionen von Dynamo zu erweitern. Sie stehen für alle Benutzer zur Verfügung und können mit einem Klick auf die Schaltfläche heruntergeladen werden.

Installieren von Dynamo-Paketen

Sie können Pakete auf der Registerkarte Nach Paketen suchen des Package Manager suchen und installieren. Verwenden Sie das Suchfeld, um nach bestimmten Paketen zu suchen. Sie können auch Filter und eine Sortierung anwenden, um die Liste entsprechend Ihren Kriterien einzugrenzen, z. B. Host-Programm oder Fehlen von Abhängigkeiten.

Die Versionskompatibilität wird über Symbole auf der Seite Nach Paketen suchen angezeigt:

  • Eine Markierung mit grünem Häkchen bedeutet, dass die Version wahrscheinlich mit Ihrer aktuellen Version von Dynamo und dem Host-Programm, wie z. B. Revit (falls vorhanden), kompatibel ist.
  • Eine Markierung mit einer roten Raute bedeutet, dass die Version wahrscheinlich nicht mit Ihrer aktuellen Version von Dynamo und dem Host (falls vorhanden) kompatibel ist.
  • Eine Markierung mit einem blauen Fragezeichen bedeutet, dass keine Kompatibilitätsinformationen für die Paketversion vorhanden sind. Sie kann möglicherweise mit Ihrem Setup verwendet werden oder auch nicht.

Um weitere Informationen zu einem Paket anzuzeigen, klicken Sie auf Info oder auf den Namen des Pakets. Dadurch wird eine Seitenleiste geöffnet, in der Sie eine Beschreibung des Pakets sowie Versions- und Lizenzinformationen und vieles mehr finden.

Um ein Paket zu installieren, klicken Sie in der Suchansicht auf Installieren. Wenn der Paketautor Kompatibilitätsinformationen zum Paket hinzugefügt hat, wird vorgabemäßig die neueste kompatible Version installiert. Andernfalls wird die neueste Version installiert, es sei denn, Sie wählen eine andere Version aus dem Dropdown-Menü aus. In der Gruppe Paketdetails können Sie oben in einer bestimmten Version im Abschnitt Versionen und Paketanforderungen auf Installieren klicken, um diese Version zu installieren.

Um den Ordner anzugeben, in dem das Paket installiert wird, klicken Sie auf den Dropdown-Pfeil neben der Schaltfläche Installieren.

Sie finden eine Liste der Pakete, die Sie installiert haben, auf der Registerkarte Installierte Pakete des Package Manager. Auf dieser Registerkarte können Sie Paketdateien anzeigen, Pakete deinstallieren und vieles mehr.

Abstimmung für Pakete

Wenn Sie Ihre Wertschätzung für die Arbeit eines Paketautors zum Ausdruck bringen möchten, können Sie für ein Paket abstimmen, das Sie installiert haben. Wechseln Sie zu Nach Paketen suchen > Info, und klicken Sie auf das Herzsymbol oben in den Paketdetails. Dies ist nur für installierte Pakete verfügbar.

Publizieren von Dynamo-Paketen

Mit dem Package Manager können Sie Pakete in Dynamo publizieren. Öffnen Sie die Registerkarte Paket publizieren, um mit dem Schritt-für-Schritt-Prozess zum Publizieren eines Pakets zu beginnen. Auf der Registerkarte Paket publizieren werden Sie aufgefordert, alle erforderlichen und optionalen Informationen zu dem Paket einzugeben, das Sie publizieren möchten.

Schritt 1: Paketdetails

Auf dieser Seite können Sie Informationen über das Paket als Ganzes sowie über die jeweilige Version hinzufügen. Nur der Paketname, die Beschreibung und die Versionsnummer sind erforderlich. Alle anderen Felder sind optional, werden aber empfohlen, um Ihren Benutzern zu helfen, den Zweck des Pakets zu erfassen und besser zu verstehen.

Paketinformationen

Name (erforderlich):

Name des Pakets, das Sie publizieren möchten. Dieser wird in der Bibliothek, in der Paketsuche und auf der Package Manager-Website angezeigt. Paketnamen müssen mindestens 3 Zeichen lang sein.

Beschreibung (erforderlich):

Die Beschreibung bietet die Gelegenheit, weitere Details zu Ihrem Paket hinzuzufügen, einschließlich der Auflistung von Ausnahmen und Voraussetzungen. Sie kann auch dazu verwendet werden, Benutzern Informationen darüber zu geben, was das Paket bietet und wie es in Arbeitsbereichen verwendet werden kann.

Für versionsspezifische Informationen ist es effizienter, einen Link zu Versionshinweisen hinzuzufügen. Beachten Sie, dass Kompatibilitätsinformationen im nächsten Schritt hinzugefügt werden, sodass Sie der Paketbeschreibung keine versionsspezifischen Kompatibilitätsinformationen hinzufügen müssen.

Gruppe:

Dieses Feld kann verwendet werden, um eine Gruppe anzugeben, zu der ein Paket gehört, z. B. die Firma/Organisation.

Schlüsselwörter:

Schlüsselwörter, die als Tags für Ihr Paket verwendet werden können, z. B. Generatives Design, ML usw. Schlüsselwörter können Benutzern helfen, Ihr Paket über die Suche zu finden. Sie können mehrere Schlüsselwörter gleichzeitig in das Feld eingeben, indem Sie sie durch ein Komma trennen und dann die EINGABETASTE drücken.

Website-Link:

Link zu einer Website, auf der der Benutzer weitere Informationen zum Paket oder zur Organisation/zum Autor erhält.

Repository-Link:

Link zum Repository des Pakets, in dem Benutzer die Paketentwicklung einsehen oder dazu beitragen können.

Urheberrechte-Inhaber:

Name der Person, die das Copyright für das Paket besitzt. Wenn Sie das Feld leer lassen, wird vorgabemäßig Ihr Benutzername verwendet.

Versionsinformationen

Version (erforderlich):

Das Versionsfeld ist in drei Abschnitte unterteilt: Hauptversion, Nebenversion und Build. Dadurch können Sie als Paketautor neue Versionen des Pakets hochladen, die der semantischen Versionierung folgen. Paketautoren möchten möglicherweise mehr Versionen für ihre Pakete mit neuen und verbesserten Funktionen publizieren, einige Verbesserungen vornehmen, die in der ersten Version nicht berücksichtigt wurden, oder einen Fehler beheben.

Link zu Versionshinweisen:

Link zu den Versionshinweisen für diese Version. Sie können jeder Version, die Sie publizieren, einen separaten Link hinzufügen.

Copyright-Jahr:

Copyright-Jahr für das Paket. Wenn Sie das Feld leer lassen, wird vorgabemäßig das Jahr verwendet, in dem das Paket publiziert wurde.

Lizenz:

Lizenz, mit der das Paket veröffentlicht wird. Die Lizenz gilt für die neueste Paketversion. Wenn Sie das Feld leer lassen, wird das Paket mit der MIT-Lizenz lizenziert.

Schritt 2: Kompatibilität

Fügen Sie Kompatibilitätsinformationen für die Paketversion hinzu. Diese Informationen helfen Benutzern, nur die Versionen Ihres Pakets zu finden und zu installieren, die mit ihrem Setup kompatibel sind. Dies reduziert potenzielle Probleme und Verwirrung. Wenn Versionskompatibilitätsinformationen zugewiesen werden, können Benutzer nach kompatiblen Versionen filtern und werden über Symbole auf der Registerkarte Nach Paketen suchen über den Kompatibilitätsstatus informiert.

Um Kompatibilitätsinformationen hinzuzufügen, wählen Sie aus, ob Ihre Paketversion nur in einem Host-Programm wie Revit oder Civil 3D, in einem Host-Programm und in Dynamo Core oder nur in Dynamo Core funktioniert. Nach der Auswahl werden zusätzliche Steuerelemente der Benutzeroberfläche angezeigt, mit denen Sie eine oder mehrere Versionen zu Ihrem Paket zuweisen können.

Bei Dynamo Core oder einem Host-Programm können Sie die Dropdown-Menüs Von und Bis verwenden, um einen Bereich auszuwählen oder Versionen einzeln in der Dropdown-Liste Einzelne Versionen hinzufügen hinzuzufügen. In der Dropdown-Liste Bis haben Sie auch die Möglichkeit, ein Platzhalterzeichen auszuwählen, z. B. 2025.x (alle Versionen). Dies bedeutet, dass die Paketversion mit allen Versionen innerhalb der angegebenen Hauptversion kompatibel ist.

Wenn Sie Versionen hinzufügen, können Sie diese der Tabelle Alle kompatiblen Versionen auf der rechten Seite hinzufügen, indem Sie in der Mitte auf die Schaltfläche Hinzufügen klicken. Wenn Sie einen Fehler gemacht haben, können Sie Versionen aus dieser Tabelle entfernen, indem Sie auf das X-Symbol in der Versionsbezeichnung klicken.

Paketautoren können Kompatibilitätsinformationen für ihre Dynamo-Pakete über die Dynamo Package Manager-Website (www.dynamopackages.com) aktualisieren. Melden Sie sich dazu auf der Website an, navigieren Sie zur Registerkarte Meine Pakete, suchen Sie Ihr Paket, und führen Sie die oben beschriebenen Schritte aus.

Schritt 3: Dateien auswählen

Dateien/Verzeichnis hinzufügen:

Wählen Sie alle Dateien aus, die mit Ihrem Paket in Verbindung stehen, und fügen Sie sie hinzu. Im Allgemeinen werden der bin-Ordner, die Binärdateien und die Abhängigkeiten hinzugefügt. Sie können einzelne Dateien oder Ordner hinzufügen.

Paketdateien auswählen:

In diesem Abschnitt können Sie den Inhalt des Pakets vor dem Publizieren organisieren. Nachdem Sie Dateien oder Ordner zum Hinzufügen ausgewählt haben, können Sie die Dateien auf der Seite Paketdateien auswählen anzeigen, entfernen oder hinzufügen. Sie können die Ordnerstruktur der ausgewählten Dateien im linken Fensterbereich und den Inhalt der einzelnen Ordner im rechten Fensterbereich anzeigen.

Um eine Datei aus dem Paket zu entfernen, klicken Sie darauf und dann auf das Papierkorb-Symbol (Entfernen) in der obersten Zeile. Sie können auf diese Weise auch mehrere Dateien auswählen und entfernen. Um ganze Ordner aus dem Paket zu entfernen, klicken Sie auf den Ordner und dann auf das Papierkorb-Symbol, das daneben angezeigt wird.

Blockbibliotheken auswählen:

Geben Sie an, welche der dem Paket hinzugefügten Dateien die Blockbibliotheken sind, indem Sie das Kontrollkästchen in der Spalte Blockbibliothek aktivieren. Blockbibliotheken enthalten die Typen und Methoden, die Dynamo als Blöcke importiert. Im Allgemeinen sollten Sie jede Assembly als Blockbibliothek markieren, die Typen enthält, die Sie in Dynamo importieren möchten. ZeroTouch-, NodeModel- oder NodeViewCustomization-Assemblys sollten als Blockbibliothek markiert werden.

Um die Benutzeroberfläche von NodeViewCustomization-Elementen zu laden, müssen diese entweder in derselben Assembly definiert sein, die die entsprechenden NodeModel-Definitionen enthält, ODER in einer separaten Assembly. In beiden Fällen muss die Assembly als Blockbibliothek markiert sein. Derzeit werden bei Assemblys, die als Blockbibliothek markiert sind und die Typen NodeModel oder NodeCustomization enthalten, alle anderen Typen ignoriert. Aus diesem Grund ist es wichtig, NodeModel-/NodeViewCustomization-Typen von den Typen zu trennen, die Sie mit dem ZeroTouch-Import importieren möchten.

Ordnerstruktur beibehalten:

Beim Erstellen eines Pakets organisiert Dynamo die Paketdateien automatisch in den Ordnern bin, doc, dyf und extra neu. Fortgeschrittene Paketautoren mit komplexen Paketordnerstrukturen können das Kontrollkästchen Ordnerstruktur beibehalten unter dem Vorschaufenster für die Paketinhalte aktivieren. Wenn diese Option aktiviert ist, nimmt Dynamo keine Änderungen an der Paketordnerstruktur vor, sondern die Paketdateien werden unverändert hochgeladen.

Pakete müssen auf eine bestimmte Weise konfiguriert werden, um sicherzustellen, dass sie funktionieren, und um Probleme zu vermeiden. Stellen Sie im Zweifelsfall sicher, dass das Kontrollkästchen deaktiviert ist, damit Dynamo die Ordnerstruktur einrichten kann.

Schritt 4: Vorschau der Dateien anzeigen

Wenn Sie mit dem Inhalt des Pakets zufrieden sind, klicken Sie auf Weiter, um eine Vorschau der Konfiguration der Paketdateien nach dem Publizieren anzuzeigen. Diese Seite ist schreibgeschützt. Um weitere Änderungen am Inhalt des Pakets vorzunehmen, gehen Sie zum vorherigen Schritt zurück.

Schritt 5: Bestätigen

Mit diesem Schritt haben Sie die Möglichkeit, alle Paketdetails, die Kompatibilität und die Dateien vor dem Publizieren anzuzeigen. Um versehentliche Änderungen zu vermeiden, sind die in den vorherigen Schritten ausgewählten Optionen auf der Seite Bestätigen schreibgeschützt. Um Änderungen daran vorzunehmen, kehren Sie zu den vorherigen Schritten zurück, indem Sie auf die Schaltfläche Zurück klicken oder die Navigationssteuerelemente oben auf der Seite verwenden.

Verzeichnis für Markdown-Datei hinzufügen (optional):

Im Abschnitt Verzeichnis für Markdown-Dateien können Sie einen Dateipfad angeben, der die Markdown-Dokumentation (.md) für Ihr Paket enthält. Die Block-Dokumentation wird im Dokumentationsbrowser angezeigt, der im oberen Navigationsmenü von Dynamo unter Erweiterungen verfügbar ist. Alternativ können Sie auch mit der rechten Maustaste auf einen Block klicken und Hilfe auswählen.

Externe Paketabhängigkeiten:

Anhand der in Schritt 3 ausgewählten Dateien erkennt Dynamo externe Paketabhängigkeiten, die hier aufgeführt sind.

Publizieren des Pakets

Nachdem Sie alle erforderlichen Informationen eingegeben haben, können Sie das Paket lokal oder online publizieren. Beim lokalen Publizieren wird das Paket im lokalen Stammverzeichnis publiziert. Beim Online-Publizieren wird das Paket in den Package Manager hochgeladen, damit es von anderen Benutzern durchsucht werden kann, sobald der Antivirenscan durchgeführt wurde. Sie finden es außerdem auf der Registerkarte Meine Pakete.

Antivirenscans

Um allen Benutzern von Dynamo-Paketen eine sichere Umgebung zu bieten, haben wir Antivirenscans für neue Pakete eingeführt. Wenn ein Benutzer ein Paket hochlädt, wird es auf Viren überprüft und ist nur dann öffentlich verfügbar, wenn es als sicher gekennzeichnet wurde. Der Scan wird während des Hochladevorgangs des Pakets durchgeführt und wirkt sich nicht auf die übliche Dauer von einigen Minuten aus, bis das neue Paket online verfügbar ist.

Wenn Ihr Paket als infiziert gekennzeichnet wird, wird auf der Package Manager-Suchseite eine Benachrichtigung angezeigt, dass das hochgeladene Paket als infiziert erkannt wurde und nicht online publiziert wird. Um das Paket öffentlich verfügbar zu machen, überprüfen Sie die im Paket enthaltenen Dateien, und laden Sie eine neue Version über den Package Manager hoch.

Verwalten von Paketen

Eine Liste Ihrer publizierten Pakete finden Sie auf der Registerkarte Meine Pakete des Package Manager, wo Sie Pakete als veraltet markieren, eine neue Version publizieren und viele weitere Aktionen durchführen können.

Markieren eines Pakets als veraltet

Als Paketeigentümer können Sie ein Paket als veraltet markieren. Dadurch wird das Paket mit einer Markierung versehen, um andere Benutzer darüber zu informieren, dass das Paket eingestellt wurde und nicht mehr aktualisiert wird. Bei der Paketsuche werden vorgabemäßig veraltete Pakete herausgefiltert. Sie können jedoch den Filter Veraltet anwenden, um sie in die Suchergebnisse aufzunehmen.

Um ein Paket als veraltet zu markieren, suchen Sie das Paket im Package Manager auf der Registerkarte Installierte Pakete oder Meine Pakete. Klicken Sie auf das Menü Optionen (drei Punkte), und wählen Sie Verwerfen aus.

Übertragen der Eigentümerschaft eines Pakets

Derzeit können Sie die Paketeigentümerschaft nicht über den Package Manager übertragen. Sie können das Dynamo-Team bitten, einen zusätzlichen Eigentümer hinzuzufügen. Beachten Sie, dass wir keine bestehenden Eigentümer entfernen, sondern nur weitere Maintainer für das Paket hinzufügen können.

Wenn Sie Ihrem vorhandenen Paket ein Eigentümer-Konto hinzufügen möchten, senden Sie eine E-Mail an dynamoteam@dynamobim.org. Stellen Sie sicher, dass Sie den Paket- und Kontonamen angeben, die Sie hinzufügen möchten.

Löschen eines Pakets

Wenn Sie einen Löschantrag stellen möchten, senden Sie eine E-Mail mit dem Betreff ‘Package Deletion Request’ an dynamoteam@dynamobim.org und geben den Paketnamen an. Sobald die Anfrage eingegangen ist, wird eine Bestätigungs-E-Mail an die E-Mail-Adresse des Paketautors gesendet, die mit dem Paket verknüpft ist. Sobald der Autor die Anfrage genehmigt hat, erfolgt die Löschung innerhalb von 30 Tagen. Antwortet der Autor nicht innerhalb von 30 Tagen, ist ein neuer Löschantrag erforderlich.

Beachten Sie, dass durch das Löschen eines Pakets der Paketname nicht für die Wiederverwendung durch neue Pakete freigegeben wird.

Paketeinstellungen

Sie können die Paketeinstellungen auf der Registerkarte Paketeinstellungen des Package Manager anzeigen, einschließlich der Download-Pfade für neue Pakete und der Paket-/Bibliothekssuchpfade. Zum Bearbeiten Ihrer Einstellungen klicken Sie auf der Registerkarte Paketeinstellungen auf den Link Voreinstellungen, um auf die Gruppe Voreinstellungen zuzugreifen.
