Package Manager

Dynamo Package Manager 是一个一站式工具集，可用于浏览、安装和发布软件包。 从顶部导航中的“软件包”>“软件包管理器”下访问它。

什么是软件包？

Dynamo 提供了大量即时可用的功能，还维护了一个可显著扩展 Dynamo 功能的丰富软件包库。 软件包是自定义节点或附加功能的集合。 Dynamo Package Manager 是社区的门户，可用于下载任何已联机发布的软件包。 这些工具集由第三方和 Dynamo 团队开发，用于扩展 Dynamo 的核心功能，可供所有人访问，只需单击相应按钮即可下载。

正在安装 Dynamo 软件包

可以在“软件包管理器”的“搜索软件包”选项卡中浏览和安装软件包。 使用搜索字段查找特定软件包。 还可以应用过滤器和排序来缩小列表范围以满足您的条件，例如主机程序或缺少依存关系。

版本兼容性通过整个“搜索软件包”页面中的图标来描述:

  • 带有绿色复选标记的标志表示版本可能与当前版本的 Dynamo 和主机程序(如 Revit，如果使用)兼容。
  • 带红色菱形的标志表示该版本可能与当前版本的 Dynamo 和主机(如果使用)不兼容。
  • 带有蓝色问号的标志表示软件包版本的兼容性信息未知。它可能适用也可能不适用于您的设置。

要查看有关软件包的详细信息，请单击“信息”或软件包的名称。 这将打开一个侧面板，您可以在其中阅读软件包的说明、查找版本和许可信息等。

要安装软件包，请在搜索视图中单击“安装”。 如果软件包作者已将兼容性信息添加到其软件包中，则默认情况下将安装最新的兼容版本。否则，将安装最新版本，除非您从下拉菜单中选择其他版本。在软件包详细信息面板中，可以在“版本和软件包要求”部分中单击特定版本顶部的“安装”以安装该版本。

要指定要安装软件包的文件夹，请单击“安装”按钮旁边的箭头下拉列表。

可以在“软件包管理器”的“已安装软件包”选项卡中找到已安装软件包的列表。 在此选项卡上，可以查看软件包文件、卸载软件包等。

为软件包投票

如果您想表达对软件包作者工作的赞赏，您可以为您已安装的软件包投票。转到“搜索软件包信息”，单击软件包详细信息顶部的心形图标。 这仅适用于已安装的软件包。

正在发布 Dynamo 软件包

Package Manager 允许在 Dynamo 中发布软件包。 打开“发布软件包”选项卡，开始发布软件包的分步过程。 “发布软件包”选项卡将要求您提供与要发布的软件包相关的所有必需和可选信息。

第 1 步: 套餐详细信息

在此页面上，您可以添加有关整个软件包以及单个版本的信息。只需要软件包名称、说明和版本号；所有其他字段都是可选的，但建议使用以帮助用户找到并更好地了解软件包的用途。

软件包信息

名称(必填):

要发布的软件包的名称。此名称将显示在库中、软件包搜索中和“软件包管理器”网站上。 软件包名称的长度必须至少为 3 个字符。

说明(必填):

通过说明，可以添加有关软件包的更多详细信息，包括列出任何例外情况和先决条件。 它还可用于教用户了解软件包的功能以及如何在他们的工作区中使用它。

对于特定于版本的信息，添加发行说明链接会更高效。请注意，兼容性信息将在下一步中添加，因此无需将特定于版本的兼容性信息添加到软件包说明中。

组:

此字段可用于指定软件包所属的组，例如公司/组织。

关键字:

可以用作与软件包相关标记的关键字，例如衍生式设计、ML 等。关键字可以帮助用户通过搜索找到您的软件包。可以一次在字段中键入多个关键字，方法是用逗号分隔它们，然后按 Enter 键。

网站链接:

指向网站的链接，该网站为用户提供有关软件包或组织/作者的详细信息。

储存库链接:

指向软件包存储库的链接，允许用户查看或参与软件包开发。

版权所有者:

软件包版权所有者的姓名。 如果留空，则默认为您的用户名。

版本信息

版本(必需):

版本字段分为三个部分: 主版本、次版本、内部版本，它允许您以软件包作者的身份上载遵循 版本控制语义的软件包的新版本。 软件包作者可能希望为其软件包发布具有新功能和改进功能的更多版本，完善第一个版本中遗漏的一些问题，或修复错误。

发行说明链接:

指向此版本发行说明的链接。您可以向发布的每个版本添加单独的链接。

版权年份:

软件包的版权年份。如果留空，则默认为软件包的发布年份。

许可:

发布软件包所依据的许可。 许可证适用于最新的软件包版本。如果留空，该软件包将根据 MIT 获得许可。

第 2 步: 兼容性

添加软件包版本的兼容性信息。 此信息将帮助用户查找并仅安装与其安装兼容的软件包版本，从而减少潜在问题和混淆。分配版本兼容性信息后，用户只能按兼容版本进行筛选，并通过“软件包搜索”选项卡中的图标通知兼容性状态。

要开始添加兼容性信息，请选择软件包版本是否仅适用于主机程序(如 Revit 或 Civil 3D)；适用于主机程序和 Dynamo Core；还是仅适用于 Dynamo Core。 选择后，您将看到其他 UI 控件，以开始将一个或多个版本指定给软件包。

对于 Dynamo Core 或主机程序，可以使用“从”和“到”下拉列表选择范围，或者在“添加单个版本”下拉列表中逐个添加版本。 在“到”下拉列表中，您还可以选择通配符，例如 2025.x (所有版本)。 这意味着软件包版本与该主发行版中的所有版本兼容。

添加版本时，可以通过单击中间的“添加”按钮将其添加到右侧的“所有兼容版本”表中。 如果出现错误，可以通过单击版本标记上的 x 图标从此表中删除版本。

软件包作者可以通过 Dynamo 软件包管理器网站 (www.dynamopackages.com) 更新其 Dynamo 软件包的兼容性信息。 为此，请登录到网站，导航到“我的程序包”选项卡，找到您的程序包，然后按照上述步骤作。

第 3 步: 选择文件

添加文件/目录:

选择并添加与软件包有关的所有文件。 通常，添加 bin 文件夹、二进制文件和依存关系。 可以添加单个文件或文件夹。

选择软件包文件:

此部分允许您在发布之前组织软件包内容。选择要添加的文件或文件夹后，可以在“选择包文件”页上查看、删除或添加文件。 可以在左侧面板中查看选定文件的文件夹结构，在右侧面板中查看每个文件夹的内容。

要从软件包中删除文件，请单击该文件，然后单击顶行中的垃圾桶(删除)图标。 您也可以通过这种方式选择和删除多个文件。要从软件包中删除整个文件夹，请单击该文件夹，然后单击它旁边显示的垃圾桶图标。

选择节点库:

通过选中“节点库”列中的框来指定添加到软件包中的哪些文件是节点库。 节点库包含 Dynamo 将作为节点输入的类型和方法。 通常，应该将任何组件标记为节点库，其中包含要输入到 Dynamo 中的类型。 ZeroTouch、NodeModel 或 NodeViewCustomization 组件应标记为节点库。

要加载 UI NodeViewCustomizations，它们应该在包含相应 NodeModel 定义的同一组件中进行定义，或者在单独的组件中进行定义。 在任一情况下，该组件都必须标记为节点库。 当前，对于标记为包含 NodeModel 或 NodeCustomization 类型的节点库的组件，将忽略所有其他类型。 因此，请务必将 NodeModel/NodeViewCustomization 类型与要使用 ZeroTouch 输入进行输入的类型分开。

保留文件夹结构:

创建软件包时，Dynamo 会自动将软件包文件重新组织为 bin、doc、dyf 和 extra 文件夹。具有复杂软件包文件夹结构的高级软件包作者可能需要选中软件包内容预览面板下方的“保留文件夹结构”复选框。 启用此选项后，Dynamo 不会更改软件包文件夹结构，而是将按原样上传软件包文件。

需要以某种方式设置软件包，以确保它们正常工作并避免任何问题。如有疑问，请确保取消选中该复选框，以允许 Dynamo 设置文件夹结构。

第 4 步: 预览文件

当您对软件包内容满意后，单击“下一步”以查看软件包文件在发布后将如何设置的预览。此页面仅供查看；要对软件包内容进行其他更改，请返回上一步。

第 5 步: 确认

此步骤使您有机会在发布之前查看所有软件包详细信息、兼容性和文件。为避免意外更改，在先前步骤中选择的选项在“确认”页上为只读。若要对它们进行更改，请使用“后退”按钮或页面顶部的导航控件返回到前面的步骤。

添加 Markdown 文件路径(可选):

可以选择在“Markdown 文件路径”部分中指定托管软件包的 Markdown (.md) 文档的文件路径。 节点文档将显示在文档浏览器中(位于 Dynamo 顶部导航菜单中的“扩展”下)，或者通过在节点上单击鼠标右键并选择“帮助”来访问。

外部软件包从属关系:

根据在第 3 步中选择的文件，Dynamo 会检测外部软件包从属关系，并在此处列出。

发布软件包

填写所有必需的详细信息后，您可以在本地或在线发布软件包。在本地发布时，软件包将发布在本地根目录下。联机发布时，该软件包会上传到“软件包管理器”，以供其他用户在清除防病毒扫描后进行搜索。 您还可以在“我的软件包”选项卡中找到它。

防病毒扫描

为了向所有 Dynamo 软件包用户提供安全的环境，我们已针对新软件包引入了防病毒扫描。 当用户上传软件包时，将扫描软件包以查找病毒，并且仅在软件包标记为安全后才会公开提供。 扫描将在软件包上载过程中进行，不会增加到通常新软件包联机可用的几分钟持续时间。

如果软件包标记为被感染，则 Package Manager 搜索页面会显示一条通知，指出上传的软件包发现被感染，无法联机发布。 要使软件包公开可用，请检查软件包中包含的文件，然后通过 Package Manager 上传新版本。

管理软件包

可以在“软件包管理器”的“我的软件包”选项卡中找到已发布软件包的列表，从中可以弃用软件包、发布新版本等。

弃用软件包

作为软件包所有者，您可以弃用软件包。这会在软件包上放置一个标记，以通知其他用户该软件包已停用且不再维护。 默认情况下，软件包搜索会过滤掉已弃用的软件包，但您可以应用“已弃用”过滤器以将其包含在搜索结果中。

要弃用某个软件包，请在“软件包管理器”中的“已安装软件包”或“我的软件包”选项卡下找到该软件包。 单击“选项”菜单(三个点)，然后选择“弃用”。

转移软件包的所有权

当前，无法通过“软件包管理器”转移软件包所有权。 您可以要求 Dynamo 团队添加其他所有者。请注意，我们不能删除现有的所有者，只能添加更多软件包的维护者。

如果您希望将帐户作为所有者添加到现有软件包，请发送电子邮件至 dynamoteam@dynamobim.org。确保提供要添加的软件包名称和帐户名称。

删除软件包

如果您希望提交删除请求，请发送电子邮件至 dynamoteam@dynamobim.org，主题行为“软件包删除请求”，并确保提供软件包名称。收到请求后，将向与软件包关联的软件包作者的电子邮件发送一封确认电子邮件。 作者批准请求后，将在 30 天内删除。如果作者在 30 天内未回复，则需要提出新的删除请求。

请注意，删除软件包不会释放软件包名称以供新软件包重用。

软件包设置

可以在“软件包管理器”的“软件包设置”选项卡中查看与软件包有关的设置，包括新的软件包下载路径和软件包/库搜索路径。 要编辑设置，请单击“软件包设置”选项卡中的“首选项”链接以访问“首选项”面板。
