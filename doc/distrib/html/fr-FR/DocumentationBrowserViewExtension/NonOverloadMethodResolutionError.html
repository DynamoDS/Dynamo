Attend le ou les types d'argument, mais a été appelé avec

Ce message d'erreur indique que le noeud que vous utilisez attend un certain type d'entrée mais a reçu un autre type d'entrée qu'il ne peut pas utiliser. Il est généralement suivi d'informations supplémentaires concernant les types transmis au noeud que vous utilisez et les types que le noeud doit recevoir. Voici quelques-unes des opérations les plus courantes qui déclenchent cette erreur:

  • Element.SetParameterByName: essayez d'utiliser le noeud Element.Parameters pour vérifier les paramètres dans l'élément. Consultez d'autres billets publiés sur le forum Dynamobim sur le sujet "expects argument type setparameter" (attend le type d'argument setparameter).
  • Geometry.BoundingBox: essayez d'utiliser Element.BoundingBox à la place, comme décrit ici. Consultez d'autres billets publiés sur le forum Dynamobim sur le sujet "expects argument type geometry" (attend le type d'argument geometry).

Comment corriger cette erreur?

La section ci-dessus indique certaines mesures que vous pouvez mettre en place pour corriger cette erreur. En général, vérifiez les types d'entrées qui sont attendus par le noeud que vous utilisez. Si possible, modifiez les entrées transmises au noeud afin qu'elles correspondent aux types attendus ou vérifiez si d'autres noeuds peuvent être utilisés pour obtenir la fonctionnalité souhaitée. Notez que les types d'entrées peuvent être vérifiés à l'aide du noeud Object.Type.
