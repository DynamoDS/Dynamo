Package Manager

Dynamo Package Manager — это универсальный инструмент для просмотра, установки и публикации пакетов. Доступ к нему можно получить на верхней панели навигации в разделе Пакеты — Package Manager.

Что такое пакет?

Dynamo предоставляет внушительный набор готовых функций, а также обширную библиотеку пакетов, что существенно расширяет возможности Dynamo. Пакет — это набор пользовательских узлов или дополнительных функциональных возможностей. Dynamo Package Manager — это портал сообщества, где можно скачать любой пакет, опубликованный в интернете. Это инструментарии, которые создаются как сторонними разработчиками, так и командой разработчиков Dynamo для расширения основных функций Dynamo. Они доступны всем желающим, и их можно скачать одним нажатием.

Установка пакетов Dynamo

Искать и устанавливать пакеты можно на вкладке Поиск пакетов в Package Manager. Чтобы найти определенные пакеты, используйте поле поиска. Можно также применить фильтры и сортировку, чтобы сузить список в соответствии с критериями, например, управляющая программа или нехватка зависимостей.

Совместимость версий отражена соответствующими значками на странице «Поиск пакетов»:

  • Флаг с зеленой галочкой означает, что версия, скорее всего, совместима с текущей версией приложения Dynamo и основной программы, например Revit (если она используется).
  • Флаг с красным ромбом означает, что версия, скорее всего, несовместима с текущей версией приложения Dynamo и основной программы (если она используется).
  • Флаг с синим вопросительным знаком означает, что сведения о совместимости для данной версии пакета отсутствуют. Он может как работать с вашей установкой, так и не работать.

Чтобы просмотреть дополнительные сведения о пакете, щелкните Сведения или имя пакета. Откроется боковая панель, на которой можно прочитать описание пакета, узнать версию и получить информацию о лицензии, а также получить другие сведения.

Чтобы установить пакет, нажмите Установить на виде поиска. Если разработчик добавил сведения о совместимости для своего пакета, по умолчанию устанавливается последняя совместимая версия. В противном случае устанавливается последняя версия, если только в раскрывающемся меню не будет выбрана другая версия. На панели сведений о пакете можно нажать Установить в верхней части блока определенной версии в разделе «Требования к версиям и пакету», чтобы установить эту версию.

Чтобы указать папку, в которую будет установлен пакет, щелкните стрелку раскрывающегося списка рядом с кнопкой «Установить».

Список установленных пакетов представлен на вкладке Установленные пакеты в Package Manager. На этой вкладке можно просмотреть файлы пакетов, удалить пакеты и выполнить другие действия.

Голосование за пакеты

Если вы хотите выразить признательность разработчику, можно проголосовать за установленный вами пакет. Перейдите к разделу Поиск пакетов — Сведения и щелкните значок в виде сердца в верхней части блока сведений о пакете. Доступно только для установленных пакетов.

Публикация пакетов Dynamo

Package Manager позволяет публиковать пакеты в Dynamo. Откройте вкладку Публикация пакета, чтобы начать пошаговый процесс публикации пакета. На вкладке «Публикация пакета» предлагается указать всю необходимую и дополнительную информацию, относящуюся к пакету, который требуется опубликовать.

Шаг 1: сведения о пакете

На этой странице можно добавить информацию как о пакете в целом, так и об отдельной версии. Требуются только имя пакета, описание и номер версии. Остальные поля являются необязательными, но рекомендуется их заполнить, чтобы помочь пользователям найти пакет и лучше понять его назначение.

Сведения о пакете

Имя (обязательно)

Имя пакета, который требуется опубликовать. Это имя будет отображаться в библиотеке, в результатах поиска пакетов и на веб-сайте Package Manager. Имя пакета должно содержать не менее 3 символов.

Описание (обязательно)

В описании можно указать дополнительные сведения о пакете, включая список исключений и предварительных требований. Здесь также можно рассказать о том, что делает пакет и как его использовать в рабочих пространствах.

В отношении информации о конкретной версии лучше добавить ссылку на примечания к выпуску. Обратите внимание на то, что сведения о совместимости добавляются на следующем шаге, поэтому не следует добавлять в описание пакета сведения о совместимости для конкретной версии.

Группа

В этом поле можно указать группу, к которой принадлежит пакет, например компанию или организацию.

Ключевые слова

Ключевые слова, которые можно использовать в качестве тегов, относящихся к данному пакету, например «генеративное проектирование», ML и т. д. По ключевым словам пользователям будет легче найти пакет. В поле можно ввести сразу несколько ключевых слов через запятую, а затем нажать клавишу ENTER.

Ссылка на веб-сайт

Ссылка на веб-сайт, на котором можно найти больше сведений о пакете, организации или разработчике.

Ссылка на репозиторий

Ссылка на репозиторий пакета, с помощью которой пользователи могут узнать о разработке пакета или поучаствовать в ней.

Правообладатель

Имя лица, которое обладает авторскими правами на пакет. Если оставить это поле пустым, по умолчанию будет использовано имя пользователя.

Сведения о версии

Версия (обязательно)

Поле «Версия» разделено на три раздела: основная, вспомогательная и сборка. Это позволяет разработчику пакета выгружать новые версии, которые соответствуют семантике управления версиями. Разработчики пакетов могут публиковать дополнительные версии своих пакетов с новыми и улучшенными функциями после доработки первой версии или исправления ошибки.

Ссылка на примечания к выпуску

Ссылка на примечания к этой версии. Для каждой публикуемой версии можно добавить отдельную ссылку.

Год авторского права

Год авторского права для пакета. Если оставить это поле пустым, по умолчанию будет указан год публикации пакета.

Лицензия

Лицензия, на основании которой был выпущен пакет. Лицензия применяется к последней версии пакета. Если оставить это поле пустым, лицензия на пакет будет предоставляться по MIT.

Шаг 2: совместимость

Добавьте сведения о совместимости для версии пакета. Эта информация поможет пользователям найти и установить только те версии пакета, которые будут совместимы с их установкой, что сократит риск возникновения проблем и путаницы. При назначении сведений о совместимости пользователи могут выполнять фильтрацию только по совместимым версиям и узнавать о совместимости с помощью значков на вкладке «Поиск пакетов».

Перед добавлением сведений о совместимости укажите, будет ли версия пакета работать только в основной программе, например Revit или Civil 3D, в основной программе и Dynamo Core или только в Dynamo Core. После выбора отобразятся дополнительные элементы управления пользовательского интерфейса, позволяющие назначить пакету одну или несколько версий.

Для Dynamo Core или основной программы можно задать диапазон в раскрывающихся списках От и До или добавить версии по одной в раскрывающемся списке Добавить отдельные версии. В раскрывающемся списке До также можно использовать подстановочный знак, например 2025.x (все версии). Это означает, что версия пакета совместима со всеми версиями данного основного выпуска.

По мере появления версий можно добавлять их в таблицу Все совместимые версии справа, нажав кнопку Добавить посередине. Если была допущена ошибка, можно удалить версии из этой таблицы, щелкнув значок «x» на теге версии.

Разработчики пакетов могут обновлять сведения о совместимости своих пакетов Dynamo через веб-сайт Dynamo Package Manager (www.dynamopackages.com). Для этого войдите на веб-сайт, перейдите на вкладку «Мои пакеты», найдите свой пакет и выполните действия, описанные выше.

Шаг 3: выбор файлов

Добавление файлов/папки

Выберите и добавьте все файлы, связанные с пакетом. Обычно это папка bin, двоичные файлы и зависимости. Можно добавить отдельные файлы или папки.

Выбор файлов пакета

Этот раздел позволяет упорядочить содержимое пакета перед публикацией. После выбора файлов или папок для добавления можно просмотреть, удалить или добавить файлы на странице Выбор файлов пакета. На панели слева можно просмотреть структуру папок выбранных файлов, а на панели справа — содержимое каждой папки.

Чтобы удалить файл из пакета, щелкните его, а затем щелкните значок корзины (Удалить) в верхней строке. Таким образом можно выбрать и удалить несколько файлов. Чтобы удалить папки целиком из пакета, щелкните сначала папку, а затем значок корзины рядом с ней.

Выбор библиотек узлов

Укажите, какие из добавленных в пакет файлов являются библиотеками узлов, установив флажок в столбце Библиотека узлов. Библиотеки узлов содержат типы и методы, которые Dynamo импортирует как узлы. Обычно в качестве библиотеки узлов можно отметить любую сборку, содержащую типы, которые требуется импортировать в Dynamo. Сборки ZeroTouch, NodeModel или NodeViewCustomization должны быть отмечены как библиотеки узлов.

Для загрузки сборок NodeViewCustomization пользовательского интерфейса они должны быть определены в той же сборке, которая содержит соответствующие определения NodeModel, ИЛИ определены в отдельной сборке. В любом случае сборка должна быть помечена как библиотека узлов. В настоящее время для сборок, помеченных как библиотеки узлов, которые содержат типы NodeModel или NodeCustomization, все другие типы игнорируются. Поэтому важно отделить типы NodeModel/NodeViewCustomization от типов, которые требуется импортировать с помощью импорта ZeroTouch.

Сохранение структуры папок

При создании пакета приложение Dynamo автоматически реорганизует файлы пакета в папки bin, doc, dyf и extra. Опытным разработчикам пакетов со сложной структурой папок может потребоваться установить флажок Сохранить структуру папок под панелью предварительного просмотра содержимого пакета. Если этот параметр включен, Dynamo не изменяет структуру папок пакета, файлы пакета выгружаются как есть.

Пакеты должны быть настроены определенным образом, чтобы обеспечить их работоспособность и предотвратить возникновение проблем. Если сомневаетесь, убедитесь, что флажок снят, чтобы разрешить Dynamo настроить структуру папок.

Шаг 4: предварительный просмотр файлов

Если содержимое пакета вас устраивает, нажмите кнопку «Далее», чтобы проверить, какой будет структура файлов пакета после его публикации. Эта страница доступна только для просмотра. Чтобы внести изменения в содержимое пакета, вернитесь к предыдущему шагу.

Шаг 5: подтверждение

На этом этапе можно проверить все сведения о пакете, совместимость и файлы перед публикацией. Во избежание случайных изменений параметры, выбранные на предыдущих этапах, на странице подтверждения доступны только для чтения. Для внесения изменений вернитесь к предыдущим шагам с помощью кнопки «Назад» или элементов управления навигацией в верхней части страницы.

Добавление пути к файлу разметки (необязательно)

В разделе Путь к файлам разметки можно указать путь к файлу, который содержит документацию по разметке (MD) для пакета. Документация по узлам отображается в обозревателе документации. Чтобы получить к нему доступ, перейдите в раздел Расширения в верхнем меню навигации Dynamo или нажмите узел правой кнопкой мыши и выберите Справка.

Внешние зависимости пакета

На основе файлов, выбранных на шаге 3, программа Dynamo определяет внешние зависимости пакета, которые приводятся на этой странице.

Публикация пакета

После указания всех необходимых данных пакет можно опубликовать на локальных ресурсах или в интернете. В первом случае пакет публикуется в локальном корневом каталоге. Во втором — пакет выгружается в Package Manager и становится доступным для поиска другими пользователями после прохождения антивирусной проверки. Его также можно найти на вкладке Мои пакеты.

Антивирусное сканирование

Для обеспечения безопасной работы пользователей с пакетами Dynamo мы внедрили антивирусное сканирование для новых пакетов. Когда пользователь выгружает пакет, он проверяется на наличие вирусов и становится общедоступным, только если была подтверждена его безопасность. Сканирование выполняется в процессе выгрузки пакета, не увеличивая его продолжительность. Через несколько минут пакеты становятся доступны в интернете.

Если будет обнаружен вирус, на странице поиска Package Manager появится уведомление о том, что выгруженный пакет заражен и не будет опубликован в интернете. Чтобы сделать пакет общедоступным, проверьте файлы, включенные в пакет, и выгрузите новую версию с помощью Package Manager.

Управление пакетами

Список опубликованных пакетов представлен на вкладке Мои пакеты в Package Manager, где можно исключить пакеты, опубликовать новую версию и т. д.

Исключение пакета

Как владелец пакета вы можете исключить его. При этом пакет помечается тегом, обозначающим, что выпуск пакета прекращен и он больше не поддерживается. Во время поиска исключенные пакеты отфильтровываются по умолчанию, хотя вы можете применить фильтр «Исключено», чтобы включить такие пакеты в результаты поиска.

Чтобы исключить пакет, найдите его в Package Manager на вкладке Установленные пакеты или Мои пакеты. Щелкните меню «Параметры» (три точки) и выберите «Исключить».

Передача права владения пакетом

В настоящее время право владения пакетом невозможно передать через Package Manager. Можно попросить разработчиков Dynamo добавить дополнительного владельца. Обратите внимание на то, что не получится удалить существующих владельцев, можно только добавить специалистов по обслуживанию пакета.

Если вы хотите добавить учетную запись в качестве владельца существующего пакета, отправьте сообщение на адрес dynamoteam@dynamobim.org. Укажите название пакета и имя учетной записи, которую требуется добавить.

Удаление пакета

Если вы хотите подать запрос на удаление, отправьте сообщение с темой «Запрос на удаление пакета» на адрес dynamoteam@dynamobim.org, указав в письме название пакета. После получения запроса разработчику отправляется сообщение для подтверждения на связанный с пакетом адрес электронной почты. После того как разработчик подтвердит запрос, пакет будет удален в течение 30 дней. Если разработчик ничего не ответит в течение 30 дней, потребуется новый запрос на удаление.

Обратите внимание на то, что удаление пакета не освобождает его название для повторного использования в новых пакетах.

Параметры пакета

Параметры пакета, включая новые пути для скачивания и пути поиска пакетов/библиотек, доступны на вкладке Параметры пакета в Package Manager. Чтобы изменить параметры, щелкните ссылку «Настройки» на вкладке «Параметры пакета» для перехода к соответствующей панели.
