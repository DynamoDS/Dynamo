## Подробности
В приведенном ниже примере Т-сплайновая поверхность становится недопустимой, что можно заметить по перекрывающимся граням при фоновом просмотре. Недопустимость поверхности подтверждается неудачной попыткой активировать режим сглаживания с помощью узла `TSplineSurface.EnableSmoothMode`. Кроме того, узел `TSplineSurface.IsInBoxMode` возвращает значение `true`, даже если поверхность изначально активировала режим сглаживания.

Для восстановления поверхность проходит через узел `TSplineSurface.Repair`. Результатом является допустимая поверхность, в чем можно убедиться, успешно активировав предварительный просмотр в режиме сглаживания.
