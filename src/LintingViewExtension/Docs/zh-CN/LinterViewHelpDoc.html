图形状态

概述

在“图形状态”扩展中，可以找到 Dynamo 在图形中检测到的问题的相关信息。 “图形状态”面板中显示的问题与工作空间中各个 Dynamo 节点上显示的警告和错误是分开的。 相反，它们源自可以由 Dynamo 扩展定义的自定义规则。

图形类型

当 Dynamo 扩展利用 Dynamo 的验证 API 注册一组自定义规则时，该扩展的名称将作为选项显示在标有“图形类型”的下拉列表中。 每当对图形进行更改时，Dynamo 都会检查这些规则是否已被破坏。

例如，适用于 Dynamo 的衍生式设计扩展使用此功能，来确保设计用于衍生式设计的 Dynamo 图形具有该图形在衍生式设计应用程序中正常工作所需的必要特性。 衍生式设计要求图形至少有一个输入节点和一个输出节点，这是使用 Dynamo 扩展注册的规则强制执行的。

节点问题和图形问题

由扩展定义的规则注册为以下两种类型之一: 图形规则或节点规则。 如果 Dynamo 在当前图形中检测到这些规则之一已被破坏，则会在这两个部分之一中显示警告(其中包含有关如何解决问题的信息)以及指向检测到问题所在位置的链接。

其他信息

每当保存图形时，“图形状态”扩展都会将一些元数据存储在 Dynamo 文件中，例如选定的图形类型以及上次保存图形时是否存在问题。

创建一个新 Dynamo 图形时，“图形状态”扩展不会自动检查任何特定规则。 “图形类型”的默认选择将为“无”。 当前，在“图形状态”视图中一次只能选择一个图形类型。
