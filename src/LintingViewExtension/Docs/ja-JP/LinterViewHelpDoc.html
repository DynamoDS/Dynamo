グラフ ステータス

概要

グラフ ステータス拡張機能では、Dynamo がグラフ内で検出した問題に関する情報を参照できます。 [グラフ ステータス]パネルに表示される問題は、ワークスペース内の個々の Dynamo ノードに表示される警告やエラーとは別のものです。 それらは、Dynamo 拡張機能で定義可能な、カスタム ルールに由来した問題です。

グラフ タイプ

Dynamo の拡張機能で Dynamo のリンティング API を使用してカスタムのルール セットを登録すると、その拡張機能の名前が[グラフ タイプ]ドロップダウンのオプションとして表示されます。 グラフに変更が加えられるたびに、Dynamo はこれらのルールが破られていないか確認します。

たとえば Dynamo のジェネレーティブ デザイン拡張機能は、この機能を使用して、ジェネレーティブ デザインで使用するために設計された Dynamo グラフに、ジェネレーティブ デザイン アプリケーションで正しく動作するために必要なプロパティがあるかを確認します。 ジェネレーティブ デザインでは、グラフに少なくとも 1 つの入力ノードと 1 つの出力ノードが必要です。これは、Dynamo 拡張機能で登録されたルールを使用して適用されます。

ノードの問題とグラフの問題

拡張機能によって定義されたルールは、グラフ ルールまたはノード ルールのいずれかとして登録されます。 現在のグラフでルールが破られていることを Dynamo が検出すると、問題の修正方法を記載した警告と、問題が検出された場所へのリンクが、これら 2 つのセクションのいずれかに表示されます。

その他の情報

グラフが保存されるたびに、グラフ ステータス拡張機能は、選択したグラフ タイプや、グラフを最後に保存したときに問題が存在していたかどうかなどのメタデータを Dynamo ファイルに保存します。

新しい Dynamo グラフを作成する時に、グラフ ステータス拡張機能が特定のルールを自動的にチェックすることはありません。 グラフ タイプの既定の選択は[なし]です。 現在、グラフ ステータス ビューでは、一度に 1 つのグラフ タイプのみを選択できます。
