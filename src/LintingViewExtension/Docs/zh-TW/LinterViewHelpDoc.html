圖表狀態

概述

「圖表狀態」延伸可讓您尋找 Dynamo 在圖表中偵測到之問題的相關資訊。「圖表狀態」面板中展示的問題與工作區中個別 Dynamo 節點顯示的警告和錯誤不同。這些問題是從可由 Dynamo 延伸定義的自訂規則衍生。

圖表類型

當 Dynamo 延伸使用 Dynamo 的 Lint 分析 API 註冊自訂規則集時，該延伸的名稱在標示為圖表類型的下拉式清單中會顯示為一個選項。每當圖表發生變更時，Dynamo 會檢查是否違反了這些規則。

例如，Dynamo 的「衍生式設計」延伸會使用此功能，確保針對「衍生式設計」而設計的 Dynamo 圖表有必要的性質，以便圖表能在「衍生式設計」應用程式中正常運作。「衍生式設計」要求圖表至少要有一個輸入節點和一個輸出節點，這會利用 Dynamo 延伸所註冊的規則強制執行。

節點問題和圖表問題

由延伸定義的規則會註冊為兩種類型之一: 圖表規則或節點規則。 如果 Dynamo 偵測到目前圖表違反了其中一個規則，則會在這兩個區段中的其中一個顯示警告，其中包含如何修正問題的資訊，以及一個連結導向偵測到問題的位置。

其他資訊

每當儲存圖表時，「圖表狀態」延伸會將某些詮釋資料儲存在 Dynamo 檔案中，例如選取的圖表類型，以及上次儲存圖表時是否出現問題。

建立新 Dynamo 圖表時，「圖表狀態」延伸不會自動檢查任何特定規則。「圖表類型」的預設選項為無。「圖表狀態」視圖中目前一次只能選取一個圖表類型。
