Package Manager

Le Package Manager Dynamo est un point d'accès unique pour la navigation, l'installation et la publication de packages. Accédez-y à partir de la navigation supérieure sous Packages Package Manager.

Que sont les packages ?

Dynamo offre un grand nombre de fonctionnalités prêtes à l'emploi et dispose également d'une bibliothèque de packages complète qui permet d'étendre considérablement les capacités de Dynamo. Un package est un ensemble de noeuds personnalisés ou de fonctionnalités supplémentaires.Le Package Manager Dynamo est un portail permettant à la communauté de télécharger tout package publié en ligne. Ces jeux d'outils sont développés par des tiers pour étendre les fonctionnalités clés de Dynamo, accessibles à tous, et prêts à être téléchargés en un clic.

Installation des packages Dynamo

Vous pouvez parcourir et installer les packages dans l'onglet Rechercher des packages du Package Manager. Utilisez le champ de recherche pour localiser des packages spécifiques. Vous pouvez également appliquer des filtres et des options de tri pour affiner la liste selon vos critères, par exemple un programme hôte ou un manque de dépendances.

Pour afficher plus d'informations sur un package, cliquez sur Afficher les détails. Un panneau s'ouvre, dans lequel vous pouvez lire une description du package, rechercher des informations sur la version et la licence, et plus encore.

Pour installer un package, cliquez sur Installer dans la vue de recherche ou dans le volet d'informations.Vous pouvez spécifier une version à installer dans la vue de recherche en cliquant sur la liste déroulante et en sélectionnant une version. Dans le cas contraire, la version la plus récente sera installée.Dans le volet d'informations sur le package, cliquez sur Installer sous une version spécifique dans le tableau des versions et de la configuration requise pour installer cette version.

Vous trouverez la liste des packages installés dans l'onglet Packages installés du Package Manager. Cet onglet permet d'afficher les fichiers de package, de désinstaller des packages, etc.

Publication de packages Dynamo

Le Package Manager vous permet de publier des packages dans Dynamo. Ouvrez l'onglet Publier un package pour commencer.L'onglet Publier un package vous demande toutes les informations obligatoires et facultatives relatives au package que vous souhaitez publier.

Ajout de détails sur le package

Nom du package :
Nom du package que vous souhaitez publier. Il apparaîtra dans la bibliothèque, dans la recherche de package et sur le site Web du Package Manager. Les noms de package doivent comporter au moins 3 caractères.

Description:
La description est un excellent endroit pour ajouter plus de détails sur votre package, y compris la liste des exceptions et des conditions préalables. Elle peut également être utilisée pour informer les utilisateurs sur le fonctionnement du package et sur la manière dont il peut être utilisé dans leurs espaces de travail.

Version:
Le champ Version est divisé en trois sections : Major, minor et build, ce qui vous permet, en tant qu'auteur du package, de charger de nouvelles versions de votre package qui suivent la sémantique du contrôle des versions. Les auteurs de packages peuvent vouloir publier plus de versions pour leurs packages avec des fonctionnalités nouvelles et améliorées, peaufiner certains coins qui ont été laissés dans la première version, ou réparer un bogue.

Hôte :
Si votre package repose sur un programme hôte, tel que Revit, Civil 3D, Alias ou d'autres, vous pouvez sélectionner un ou plusieurs hôtes dans la liste déroulante.Lors de la recherche de packages, les utilisateurs peuvent filtrer les résultats en fonction de l'hôte pour les aider à localiser les packages conçus pour fonctionner avec leurs programmes préférés.

Mots-clés (facultatif):
Mots-clés pouvant être utilisés comme balises associées à votre package, par exemple Generative Design, ML, etc. Vous pouvez entrer plusieurs mots-clés dans le champ à la fois en les séparant par la touche espace, puis en appuyant sur la touche Entrée.

Groupe (facultatif) :
Ce champ peut servir à spécifier le groupe auquel un package appartient, tel que la société/l'organisation qui l'a développé.

Site Web (facultatif) :
Lien vers un site Web qui peut fournir à l'utilisateur des détails supplémentaires sur le package ou l'organisation/auteur.

URL du référentiel (facultatif):
Un lien vers le répertoire du package qui permet aux utilisateurs de voir ou de contribuer au développement du package.

Licence (facultatif) :
Licence sous laquelle le package est publié. Licence par défaut: MIT.

Détenteur du copyright (facultatif) :
Nom de la personne détenant le copyright du package. Si ce champ n'est pas renseigné, le nom d'utilisateur par défaut est votre nom.

Année de copyright (facultatif) :
Année de copyright du package. Si cette zone n'est pas renseignée, le package est défini par défaut sur son année de publication.

Ajout de fichiers de package

Contenu du package :
Cette section, située dans la barre latérale droite de l'onglet Publier un package, vous permet d'organiser le contenu de votre package avant sa publication.

Ajouter des fichiers/répertoires :

Sélectionnez et ajoutez tous les fichiers associés à votre package. En règle générale, le dossier bin, les fichiers binaires et les dépendances sont ajoutés.Vous pouvez ajouter des fichiers ou des dossiers individuels.

Après avoir sélectionné les fichiers ou les dossiers à ajouter, vous pouvez afficher, supprimer ou ajouter des fichiers sur la page Sélectionner le contenu du package. Vous pouvez afficher la structure de dossiers des fichiers sélectionnés dans le volet de gauche et le contenu de chaque dossier dans le volet de droite.

Sélectionner les bibliothèques de noeuds:

Sélectionnez les fichiers que vous avez ajoutés au package qui sont les bibliothèques de noeuds en cochant la case dans la colonne Bibliothèque de noeuds.Les bibliothèques de noeuds contiennent les types et méthodes que Dynamo importera en tant que noeuds. En général, vous devez marquer un ensemble comme bibliothèque de noeuds contenant les types que vous souhaitez importer dans Dynamo.Les ensembles ZeroTouch, NodeModel ou NodeViewCustomization doivent être marqués comme bibliothèque de noeuds.

Pour charger les personnalisations NodeViewCustomizations de l’interface utilisateur, elles doivent être définies dans le même ensemble contenant les définitions NodeModel correspondantes OU définies dans un ensemble distinct.Dans les deux cas, l'ensemble doit être étiqueté en tant que bibliothèque de noeuds. Actuellement, pour les ensembles marqués comme bibliothèque de noeuds contenant les types NodeModel ou NodeCustomization, tous les autres types seront ignorés. Pour cette raison, il est important de séparer les types NodeModel/NodeViewCustomization des types que vous souhaitez importer à l'aide de l'importation ZeroTouch.

Conserver la structure des dossiers :

Les auteurs de packages gérant des structures de dossiers de package complexes peuvent activer le bouton bascule Conserver la structure des dossiers sous le panneau d'aperçu du contenu du package. Ce paramètre est activé par défaut lorsqu'un utilisateur publie une nouvelle version d'un package existant. Dans ce cas, Dynamo n'apporte pas de modifications à la structure de dossiers de votre package et le package est chargé en l'état.

Si vous avez besoin que votre package soit configuré d'une certaine manière pour vous assurer qu'il fonctionne et éviter tout problème, activez le paramètre. En cas de doute ou de publication d'un nouveau module, désactivez le paramètre pour permettre à Dynamo de configurer la structure de dossiers.

Aperçu du contenu du package :
Lorsque vous êtes satisfait du contenu de votre package, cliquez sur Suivant pour afficher un aperçu de la configuration des fichiers de package une fois qu'ils ont été publiés.Cette page est en lecture seule ; pour apporter des modifications supplémentaires au contenu du module, revenez à la page précédente.Cliquez sur Terminer pour revenir à la page des détails du package.

Ajouter un chemin de fichier de marquage (facultatif) :
Vous avez la possibilité de spécifier un chemin de fichier qui héberge la documentation de marquage (.md) pour votre package dans la section Chemin des fichiers de marquage.

Publication de votre package
Une fois que vous avez renseigné tous les détails requis, vous pouvez publier le package localement ou en ligne. Lors d'une publication locale, le package est publié dans le répertoire racine local, tandis que lors d'une publication en ligne, le package est téléchargé dans le Package Manager.

Analyses antivirus

Pour fournir un environnement sécurisé à tous les utilisateurs du package Dynamo, nous avons introduit des analyses antivirus pour les nouveaux packages. Lorsqu'un utilisateur charge un package, il est analysé à la recherche de virus et n'est disponible publiquement que lorsqu'il est marqué comme étant sûr. L'analyse se produit pendant le processus de chargement du package et ne s'ajoute pas à la durée habituelle de 10 à 15 minutes pour que le nouveau package soit disponible en ligne. Si votre package est signalé comme infecté, la page de recherche du Package Manager affiche une notification indiquant que le package téléchargé est infecté et ne sera pas publié en ligne. Pour rendre le package accessible au public, vérifiez les fichiers inclus dans le package et chargez une nouvelle version via le Package Manager.

Gestion de vos packages

Vous pouvez trouver une liste de vos packages publiés dans l'onglet Mes packages du Package Manager, où vous pouvez les supprimer, publier une nouvelle version, etc.

Paramètres de package

Vous pouvez afficher les paramètres relatifs au package dans l'onglet Paramètres du package du Package Manager, y compris les nouveaux chemins de téléchargement et de recherche de package/bibliothèque. Pour modifier vos paramètres, cliquez sur le lien Préférences dans l'onglet Paramètres de package pour accéder au volet Préférences.
