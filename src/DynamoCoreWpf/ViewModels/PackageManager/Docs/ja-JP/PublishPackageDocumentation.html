Package Manager

Dynamo Package Manager はパッケージを参照、インストール、パブリッシュするためのワンストップショップです。 上部ナビゲーションにある[パッケージ]の[Package Manager]からアクセスします。

パッケージとは

Dynamo には豊富な機能が用意されていますが、Dynamo の機能を大幅に拡張できる広範囲に及ぶパッケージ ライブラリも保持しています。 パッケージとは、カスタム ノードや追加機能を収集したものです。 Dynamo Package Manager は、オンラインでパブリッシュされたパッケージをダウンロードするためのコミュニティ ポータルです。 これらのツールセットは、Dynamo の基本機能を拡張するためにサードパーティと Dynamo チームの両方によって開発され、すべてのユーザがアクセスでき、ボタンをクリックするだけでダウンロードできます。

Dynamo パッケージをインストールする

Package Manager の[パッケージを検索]タブで、パッケージを検索してインストールできます。 特定のパッケージを検索するには、検索フィールドを使用します。 フィルタや並べ替えを利用すると、ホスト プログラムや依存関係の不足など、任意の条件に合わせてリストを絞り込むこともできます。

バージョンの互換性は、[パッケージを検索]ページ全体でアイコンで示されます。

  • 緑のチェックマークが付いたフラグは、そのバージョンが現在のバージョンの Dynamo および Revit (Revit を使用している場合)などのホスト プログラムと互換性がある可能性が高いことを示しています。
  • 赤いひし形が付いたフラグは、そのバージョンが現在のバージョンの Dynamo およびホスト(使用している場合)と互換性がない可能性が高いことを示しています。
  • 青い疑問符が付いたフラグは、パッケージ バージョンの互換性情報が不明であることを示しています。 設定によっては、機能する場合と機能しない場合があります。

パッケージの詳細を表示するには、[情報]またはパッケージ名をクリックします。 これにより、サイド パネルが開き、パッケージの説明を読んだり、バージョンやライセンス情報などの詳細を確認できます。

パッケージをインストールするには、検索ビューの[インストール]をクリックします。 パッケージ作成者がパッケージに互換性情報を追加した場合、互換性のある最新のバージョンが既定でインストールされます。 それ以外の場合は、ドロップダウン メニューから別のバージョンを選択しない限り、最新バージョンがインストールされます。 パッケージの詳細パネルの[バージョンとパッケージの要件]セクションで、特定バージョンの一番上にある[インストール]をクリックして、そのバージョンをインストールすることができます。

パッケージをインストールするフォルダを指定するには、[インストール]ボタンの横にある矢印ドロップダウンをクリックします。

インストールしたパッケージの一覧は、Package Manager の[インストール済みパッケージ]タブに表示されます。 このタブでは、パッケージ ファイルの表示や、パッケージのアンインストールなどができます。

パッケージの評価

パッケージ作成者の作業に感謝の意を表したい場合は、インストールしたパッケージを評価できます。 [パッケージ情報を検索]に移動し、パッケージの詳細の上部にあるハートのアイコンをクリックします。 これは、インストールされているパッケージでのみ使用できます。

Dynamo パッケージをパブリッシュする

Package Manager を使用すると、Dynamo 内でパッケージをパブリッシュすることができます。 [パッケージをパブリッシュ]タブを開いて、パッケージをパブリッシュする段階的なプロセスを開始します。 [パッケージをパブリッシュ]タブでは、パブリッシュするパッケージに関するすべての必須情報および省略可能な情報を入力するよう求められます。

手順 1: パッケージの詳細

このページでは、パッケージ全体および個々のバージョンに関する情報を追加できます。 パッケージ名、説明、バージョン番号のみが必須です。他のすべてのフィールドは省略可能ですが、ユーザがパッケージを見つけてその目的をよりよく理解できるようにしておくことをお勧めします。

パッケージ情報

名前(必須):

パブリッシュするパッケージの名前です。 これは、ライブラリ、パッケージ検索、および Package Manager の Web サイトに表示されます。 パッケージ名の長さは 3 文字以上にする必要があります。

説明(必須):

説明には、例外や前提条件の一覧表示など、パッケージの詳細を追加できます。 また、パッケージの機能とワークスペースでの使用方法についてユーザを教育するためにも使用できます。

バージョン固有の情報については、リリース ノートのリンクを追加する方が効率的です。 互換性情報は次の手順で追加されるため、バージョン固有の互換性情報をパッケージの説明に追加する必要はありません。

グループ:

このフィールドは、会社や組織など、パッケージが属するグループを指定するために使用できます。

キーワード:

パッケージに関連するタグとして使用できるキーワード(例: ジェネレーティブ デザイン、ML など)。キーワードは、ユーザが検索でパッケージを見つけるのに役立ちます。 一度に複数のキーワードをフィールドに入力するには、キーワードをカンマで区切り、[Enter]を押します。

Web サイト リンク:

パッケージまたは組織や作成者に関する詳細情報をユーザに提供する Web サイトへのリンク。

リポジトリ リンク:

ユーザがパッケージ開発を表示したり、パッケージ開発に貢献できるパッケージのリポジトリへのリンク。

著作権保有者:

パッケージの著作権を保有しているユーザの名前。 空白のままにすると、既定ではユーザ名になります。

バージョン情報

バージョン(必須):

バージョンのフィールドは、メジャー、マイナー、ビルドの 3 つのセクションに分かれています。パッケージの作成者は、バージョン管理セマンティクスに従って新しいバージョンのパッケージをアップロードできます。 パッケージ作成者は、パッケージの新しい機能や改善された機能を含むバージョンをさらにパブリッシュしたり、最初のバージョンで残された部分を改良したり、バグを修正することができます。

リリース ノートのリンク:

このバージョンのリリース ノートへのリンク。 パブリッシュするバージョンごとに個別のリンクを追加できます。

著作権取得年:

パッケージの著作権期間です。 空白の場合、パッケージがパブリッシュされた年が既定になります。

ライセンス:

パッケージがリリースされるライセンス。 ライセンスは最新のパッケージ バージョンに適用されます。 空白の場合、パッケージは MIT に基づきライセンスされます。

手順 2: 互換性

パッケージ バージョンの互換性情報を追加します。 この情報は、設定と互換性のあるパッケージのバージョンのみをユーザが見つけてインストールするのに役立ち、潜在的な問題や混乱を減らします。 バージョンの互換性情報が割り当てられている場合、ユーザは互換性のあるバージョンのみでフィルタでき、[パッケージを検索]タブのアイコンを介して互換性ステータスが通知されます。

互換性情報の追加を開始するには、パッケージ バージョンを Revit や Civil 3D などのホスト プログラムでのみ動作させるのか、ホスト プログラムと Dynamo Core で動作させるのか、Dynamo Core のみで動作させるのかを選択します。 選択すると、パッケージに 1 つまたは複数のバージョンを割り当てるための追加の UI コントロールが表示されます。

Dynamo Core またはホスト プログラムの場合、[開始]ドロップダウンと[終了]ドロップダウンを使用して範囲を選択するか、[個別のバージョンを追加]ドロップダウンでバージョンを 1 つずつ追加することができます。 [終了]ドロップダウンで、「2025.x」(すべてのバージョン)などのワイルドカードを任意で選択することもできます。 これは、パッケージ バージョンがメジャー リリース内のすべてのバージョンと互換性があることを示しています。

バージョンを追加する際、中央の[追加]ボタンをクリックして、右側の[すべての互換性のあるバージョン]テーブルに追加できます。 操作を間違えた場合は、バージョン タグの x アイコンをクリックして、このテーブルからバージョンを削除できます。

パッケージの作成者は、Dynamo Package Manager の Web サイト(www.dynamopackages.com)から Dynamo パッケージの互換性情報を更新できます。 これを行うには、Web サイトにログインし、[マイ パッケージ]タブにナビゲートしてパッケージを見つけて、上記の手順に従います。

手順 3: ファイルを選択する

ファイル/フォルダを追加する:

パッケージに関連するすべてのファイルを選択して追加します。 通常は、bin フォルダ、バイナリ、および依存関係が追加されます。 個々のファイルまたはフォルダを追加できます。

パッケージ ファイルを選択する:

このセクションでは、パブリッシュする前にパッケージの内容を整理できます。 追加するファイルまたはフォルダを選択したら、[パッケージ ファイルを選択]ページでファイルを表示、削除、追加できます。 選択したファイルのフォルダ構造は左側のパネルに、各フォルダの内容は右側のパネルに表示されます。

パッケージからファイルを削除するには、ファイルを選択してから、一番上の行にあるごみ箱(削除)アイコンをクリックします。 この方法で複数のファイルを選択して削除することもできます。 パッケージからフォルダ全体を削除するには、フォルダを選択してから、その横に表示されるごみ箱アイコンをクリックします。

ノード ライブラリを選択する:

パッケージに追加したファイルのうち、ノード ライブラリであるものを指定するには、[ノード ライブラリ]列のチェックボックスをオンにします。 ノード ライブラリには、Dynamo がノードとして読み込むタイプとメソッドが含まれています。 通常、Dynamo に読み込むタイプを含むノード ライブラリとして、任意のアセンブリをマークします。 ZeroTouch、NodeModel、または NodeViewCustomization アセンブリは、ノード ライブラリとしてマークします。

NodeViewCustomizations の UI をロードするには、対応する NodeModel 定義を含む同じアセンブリで定義するか、別のアセンブリで定義する必要があります。 いずれの場合も、アセンブリにノード ライブラリとしてタグ付けします。 現在、NodeModel または NodeCustomization タイプを含むノード ライブラリとしてマークされたアセンブリに対して、その他のすべてのタイプは無視されます。 このため、ZeroTouch 読み込みを使用して読み込むタイプから、NodeModel または NodeViewCustomization タイプを区別することが重要です。

フォルダ構造を保持する:

パッケージを作成すると、Dynamo はパッケージ ファイルを自動的に bin、doc、dyf、extra の各フォルダに再編成します。 複雑なパッケージ フォルダ構造を持つ高度なパッケージの作成者は、パッケージの内容プレビュー パネルの下にある[フォルダ構造を保持]チェックボックスをオンにすることをお勧めします。 このオプションを有効にすると、Dynamo ではパッケージ フォルダ構造が変更されず、パッケージ ファイルが現状のままでアップロードされます。

パッケージが確実に機能するように設定し、問題を回避するには、特定の方法でパッケージをセットアップする必要があります。 不明な場合は、チェックボックスをオフにして、Dynamo でフォルダ構造を設定できるようにします。

手順 4: ファイルをプレビューする

パッケージの内容に問題がなければ、[次へ]をクリックして、パブリッシュ後のパッケージ ファイルの設定のプレビューを表示します。 このページは閲覧専用であるため、パッケージの内容をさらに変更するには、前の手順に戻ります。

手順 5: 確認

この手順により、パブリッシュ前にすべてのパッケージの詳細、互換性、ファイルを確認することができます。 誤って変更してしまうことのないように、前の手順で選択したオプションは[確認]ページで読み込み専用になります。 これらを変更するには、[戻る]ボタンまたはページ上部のナビゲーション コントロールを使用して、前の手順に戻ります。

Markdown ファイル パスを追加(省略可能):

[Markdown ファイル パス]セクションでは、パッケージの Markdown (.md)ドキュメントをホストするファイル パスを指定できます。 ノードのドキュメントはドキュメント ブラウザに表示されます。ドキュメント ブラウザは、Dynamo の上部ナビゲーション メニューの[拡張機能]で利用するか、ノードを右クリックして[ヘルプ]を選択すると表示されます。

外部パッケージの依存関係:

手順 3 で選択したファイルに基づいて、ここに一覧表示されている外部パッケージの依存関係が Dynamo で検出されます。

パッケージをパブリッシュする

必要な詳細情報をすべて入力したら、パッケージをローカルまたはオンラインでパブリッシュできます。 ローカルにパブリッシュすると、パッケージはローカルのルート フォルダにパブリッシュされます。 オンラインでパブリッシュすると、パッケージは Package Manager にアップロードされます。ウイルス対策スキャンで問題がないことが確認されたら、他のユーザが検索できるようになります。 [マイ パッケージ]タブにも表示されるようになります。

ウィルス対策スキャン

すべての Dynamo パッケージ ユーザに安全な環境を提供するために、新しいパッケージに対するウィルス対策スキャンが導入されました。 ユーザがパッケージをアップロードすると、ウィルスがないかスキャンされ、安全としてフラグが設定された後にのみパブリックに使用できるようになります。 スキャンはパッケージのアップロード処理中に発生します。新しいパッケージがオンラインで使用可能になるまでに通常かかる時間(数分)が増えることはありません。

感染しているとしてパッケージにフラグが設定されると、アップロードされたパッケージが感染していたためにオンラインでパブリッシュされないことを示す通知が、Package Manager の検索ページに表示されます。 パッケージを一般に公開するには、パッケージに含まれているファイルを確認し、Package Manager を使用して新しいバージョンをアップロードします。

パッケージを管理する

Package Manager の[マイ パッケージ]タブに、パブリッシュされたパッケージのリストがあります。このタブでは、パッケージの廃止や新しいバージョンのパブリッシュなどを行うことができます。

パッケージを廃止する

パッケージ所有者は、パッケージを廃止できます。 これにより、パッケージが廃止され、保守されなくなったことを他のユーザに通知するタグがパッケージに配置されます。 パッケージ検索では、廃止されたパッケージが既定で除外されますが、[廃止済み]フィルタを適用すると検索結果に含めることができます。

パッケージを廃止するには、Package Manager の[インストール済みパッケージ]または[マイ パッケージ]タブでパッケージを見つけます。 [オプション]メニュー(3 つのドット)をクリックし、[廃止]を選択します。

パッケージの所有権を譲渡する

現在、Package Manager を使用してパッケージの所有権を譲渡することはできません。 所有者を追加するよう Dynamo チームに依頼することができます。 既存の所有者を削除することはできず、パッケージを保持するユーザを追加するだけであることに注意してください。

既存のパッケージに所有者としてアカウントを追加する場合は、dynamoteam@dynamobim.org 宛てに電子メールを送信してください。 パッケージ名と追加するアカウント名を必ず指定してください。

パッケージを削除する

削除リクエストを送信する場合は、件名を「Package Deletion Request」とし、パッケージ名を明記したメールを dynamoteam@dynamobim.org 宛てに送信してください。 リクエストが受領されると、パッケージに関連付けられたパッケージ作成者の電子メール アドレスに確認の電子メールが送信されます。 作成者がリクエストを承認すると、削除が 30 日以内に実行されます。 作成者が 30 日以内に応答しない場合は、新しい削除リクエストが必要です。

パッケージを削除しても、新しいパッケージで再利用するためにパッケージ名が解放されるわけではないことに注意してください。

パッケージの設定

新しいパッケージのダウンロード パスや、パッケージまたはライブラリの検索パスなど、パッケージ関連の設定は Package Manager の[パッケージの設定]タブで確認できます。 設定を編集するには、[パッケージの設定]タブの[基本設定]リンクをクリックして、[基本設定]パネルにアクセスします。
