Package Manager

Dynamo Package Manager はパッケージを参照、インストール、パブリッシュするためのワンストップショップです。 上部ナビゲーションの[パッケージ]配下にある[Package Manager]からアクセスします。

パッケージとは

Dynamo には豊富な機能が用意されていますが、Dynamo の機能を大幅に拡張できる広範囲に及ぶパッケージ ライブラリも保持しています。 パッケージとは、カスタム ノードや追加機能を収集したものです。 Dynamo Package Manager は、オンラインでパブリッシュされたパッケージをダウンロードするためのコミュニティ ポータルです。 これらのツールセットは、Dynamo の基本機能を拡張するためにサード パーティによって開発されたもので、すべてのユーザがアクセスでき、ボタンをクリックするだけでダウンロードできます。

Dynamo パッケージをインストールする

Package Manager の[パッケージを検索]タブで、パッケージを参照してインストールできます。 特定のパッケージを検索するには、検索フィールドを使用します。 フィルタや並べ替えを利用すると、ホスト プログラムや依存関係の不足など、任意の条件に合わせてリストを絞り込むこともできます。

パッケージの詳細を表示するには、[詳細を表示]をクリックします。 これにより、パッケージの説明、バージョンやライセンス情報などを確認できるパネルが開きます。

パッケージをインストールするには、検索ビューまたは詳細パネルで[インストール]をクリックします。 検索ビューでドロップダウンからバージョンを選択して、インストールするバージョンを指定できます。指定しない場合は、最新バージョンがインストールされます。 パッケージの詳細パネルで[バージョンとパッケージの要件]テーブルからインストールするバージョンを指定して、その下にある[インストール]をクリックします。

インストールしたパッケージのリストは、Package Manager の[インストール済みパッケージ]タブで確認できます。 このタブでは、パッケージ ファイルの表示や、パッケージのアンインストールなどができます。

Dynamo パッケージのパブリッシュ

Package Manager を使用すると、Dynamo 内でパッケージをパブリッシュすることができます。 最初に[パッケージをパブリッシュ]タブを開きます。 [パッケージをパブリッシュ]タブでは、パブリッシュするパッケージに関するすべての必須情報および省略可能な情報を入力するよう求められます。

パッケージの詳細を追加する

パッケージ名:
パブリッシュするパッケージの名前。 これは、ライブラリ、パッケージ検索、および Package Manager の Web サイトで表示されます。 パッケージ名の長さは 3 文字以上にする必要があります。

説明:
説明には、例外や前提条件のリストなど、パッケージに関する詳細を追加しておくことができます。 またパッケージの機能やワークスペースでの使用方法を追加して、ユーザへの説明にも利用できます。

バージョン:
バージョンのフィールドは、メジャー、マイナー、ビルドの 3 つのセクションに分かれています。パッケージの作成者はバージョン管理セマンティックに従って新しいバージョンのパッケージをアップロードできます。 パッケージの作成者は、パッケージの新しい機能や改善された機能を含むバージョンをさらにパブリッシュしたり、最初のバージョンで残された部分を改良したり、バグを修正することができます。

ホスト:
パッケージが Revit、Civil3D、Alias などのホスト プログラムに依存している場合は、ドロップダウンから 1 つ以上のホストを選択できます。 ユーザは、パッケージを検索する際にホストに基づいて結果をフィルタできるので、目的のプログラムで動作するよう設計されたパッケージを見つけやすくなります。

キーワード(省略可能):
パッケージに関連するタグとして使用できるキーワード(例: ジェネレーティブ デザイン、ML など)。一度に複数のキーワードをフィールドに入力するには、各キーワードをスペース キーを使って区切り、[Enter]を押します。

グループ(省略可能):
このフィールドは、パッケージが属するグループ(会社や組織など)を指定するために使用できます。

Web サイト(省略可能):
Web サイトへのリンク。パッケージまたは組織/作成者に関する詳細情報をユーザに提供します。

リポジトリ URL (省略可能):
パッケージのリポジトリへのリンク。ユーザは、パッケージ開発の確認やパッケージ開発への投稿ができます。

ライセンス(省略可能):
パッケージがリリースされるライセンス。 既定は MIT です。

著作権保有者(省略可能):
パッケージの著作権を保有するユーザの名前。 空白のままにすると、ユーザ名が既定になります。

著作権期間(省略可能):
パッケージの著作権期間。 空白のままにすると、パッケージがパブリッシュされた年が既定になります。

パッケージ ファイルを追加する

パッケージの内容:
このセクションは[パッケージをパブリッシュ]タブの右サイドバーにあり、パブリッシュする前にパッケージの内容を整理することができます。

ファイルを追加/フォルダを追加:

パッケージに関連するすべてのファイルを選択して追加します。 通常は、bin フォルダ、バイナリ、および依存関係が追加されます。 個々のファイルまたはフォルダを追加できます。

追加するファイルまたはフォルダを選択した後、[パッケージの内容を選択]ページでファイルの表示、削除、または追加ができます。 選択したファイルのフォルダ構造は左側のパネルに、各フォルダの内容は右側のパネルに表示されます。

ノード ライブラリを選択:

[ノード ライブラリ]列のチェックボックスをオンにして、パッケージに追加したファイルのどれがノード ライブラリであるかを指定します。 ノード ライブラリには、Dynamo がノードとして読み込むタイプとメソッドが含まれています。 通常、Dynamo に読み込むタイプを含むノード ライブラリとして、任意のアセンブリをマークします。 ZeroTouch、NodeModel、または NodeViewCustomization アセンブリは、ノード ライブラリとしてマークします。

NodeViewCustomizations の UI をロードするには、対応する NodeModel 定義を含む同じアセンブリで定義するか、別のアセンブリで定義する必要があります。 いずれの場合も、アセンブリにノード ライブラリとしてタグ付けします。 現在、NodeModel または NodeCustomization タイプを含むノード ライブラリとしてマークされたアセンブリに対して、その他のすべてのタイプは無視されます。 このため、ZeroTouch 読み込みを使用して読み込むタイプから、NodeModel または NodeViewCustomization タイプを区別することが重要です。

フォルダ構造を保持:

複雑なパッケージ フォルダ構造を持つパッケージの作成者は、パッケージ内容のプレビュー パネルの下にある[フォルダ構造を保持]の切り替えを有効にできます。 この設定は、ユーザが既存のパッケージの新しいバージョンをパブリッシュする際に既定でオンになっています。この場合、Dynamo ではパッケージ フォルダ構造は変更されず、パッケージは現状のままでアップロードされます。

機能を確認して問題を回避するために、パッケージを特定の方法で設定する必要がある場合は、設定をオンに切り替えます。 不明な場合や新しいパッケージをパブリッシュする場合は、設定をオフに切り替えて、Dynamo でフォルダ構造を設定できるようにします。

パッケージの内容をプレビュー:
パッケージの内容に問題がなければ[次へ]をクリックして、パブリッシュ後のパッケージ ファイルの設定のプレビューを表示します。 このページは閲覧のみです。パッケージ内容に変更を加えるには、前のページに戻ります。 [完了]をクリックして、パッケージの詳細ページに戻ります。

Markdown ファイル パスを追加(省略可能):
[Markdown ファイル パス]のセクションでは、パッケージの Markdown (.md)ドキュメントをホストするファイル パスを指定できます。

パッケージをパブリッシュする
必要な詳細情報をすべて入力したら、パッケージをローカルまたはオンラインでパブリッシュできます。 ローカルにパブリッシュすると、パッケージはローカルのルート フォルダにパブリッシュされ、オンラインでパブリッシュすると、パッケージは Package Manager にアップロードされます。

ウィルス対策スキャン

すべての Dynamo パッケージ ユーザに安全な環境を提供するために、新しいパッケージに対するウィルス対策スキャンが導入されました。 ユーザがパッケージをアップロードすると、ウィルスがないかスキャンされ、安全としてフラグが設定された後にのみパブリックに使用できるようになります。 スキャンはパッケージのアップロード処理中に発生します。新しいパッケージがオンラインで使用可能になるまでに通常かかる時間(10 ～ 15 分)が増えることはありません。 感染しているとしてパッケージにフラグが設定されると、アップロードされたパッケージが感染していたためにオンラインでパブリッシュされないことを示す通知が、Package Manager の検索ページに表示されます。 パッケージを一般に公開するには、パッケージに含まれているファイルを確認し、Package Manager を使用して新しいバージョンをアップロードします。

パッケージを管理する

Package Manager の[マイ パッケージ]タブに、パブリッシュされたパッケージのリストがあります。このタブ内で、パッケージの廃止や新しいバージョンのパブリッシュなどができます。

パッケージの設定

新しいパッケージのダウンロード パスや、パッケージまたはライブラリの検索パスなど、パッケージ関連の設定は Package Manager の[パッケージの設定]タブで確認できます。 設定を編集するには、[パッケージの設定]タブの[基本設定]リンクをクリックして、[基本設定]パネルにアクセスします。
