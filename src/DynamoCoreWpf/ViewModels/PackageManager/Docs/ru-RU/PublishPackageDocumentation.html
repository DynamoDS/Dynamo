Package Manager

Dynamo Package Manager — это универсальный инструмент для просмотра, установки и публикации пакетов. Доступ к нему можно получить на верхней панели навигации в разделе «Пакеты» > «Package Manager».

Что представляют собой пакеты?

Dynamo предоставляет внушительный набор готовых функций, а также обширную библиотеку пакетов, что существенно расширяет возможности Dynamo. Пакет — это набор пользовательских узлов или дополнительных функциональных возможностей. Dynamo Package Manager — это портал сообщества, где можно скачать любой пакет, опубликованный в интернете. Эти общедоступные инструментарии, созданные сторонними разработчиками для расширения базовых функций Dynamo, можно скачать одним щелчком мыши.

Установка пакетов Dynamo

Найти и установить требуемые пакеты можно на вкладке «Поиск пакетов» в Package Manager. Чтобы найти определенные пакеты, используйте поле поиска. Можно также применить фильтры и сортировку, чтобы сузить список в соответствии с критериями, например, управляющая программа или нехватка зависимостей.

Чтобы просмотреть дополнительные сведения о пакете, нажмите кнопку «Показать подробности». Откроется панель, на которой можно ознакомиться с описанием пакета, найти версию и информацию о лицензии, а также получить другие сведения.

Чтобы установить пакет, нажмите кнопку «Установить» в представлении поиска или на панели сведений. В представлении поиска можно указать версию для установки. Для этого щелкните раскрывающийся список и выберите требуемую версию. В противном случае будет установлена последняя версия. На панели сведений о пакете нажмите кнопку «Установить» под определенной версией в таблице «Требования к версиям и пакету», чтобы установить эту версию.

Список установленных пакетов представлен на вкладке «Установленные пакеты» в Package Manager. На этой вкладке можно просмотреть файлы пакетов, удалить пакеты и выполнить другие действия.

Публикация пакетов Dynamo

Package Manager позволяет публиковать пакеты в Dynamo. Чтобы начать работу, откройте вкладку «Публикация пакета». На вкладке «Публикация пакета» предлагается указать всю необходимую и дополнительную информацию, относящуюся к пакету, который требуется опубликовать.

Добавление сведений о пакете

Имя пакета:
Имя пакета, который требуется опубликовать. Это имя будет отображаться в библиотеке, в результатах поиска пакетов и на веб-сайте Package Manager. Имя пакета должно содержать не менее 3 символов.

Описание:
В поле «Описание» можно указать дополнительные сведения о пакете, включая список исключений и предварительных требований. Кроме того, в этот раздел можно добавить полезную информацию о функционале пакета и возможностях его использования в рабочих пространствах.

Версия:
Поле «Версия» разделено на три части: основная, вспомогательная и сборка, благодаря чему разработчики пакетов могут выгружать новые версии пакета с соблюдением семантики для контроля версий. Разработчики пакетов могут публиковать дополнительные версии пакетов с новыми и усовершенствованными функциями, дорабатывать некоторые несовершенства первой версии или исправлять ошибки.

Основа:
Если для работы пакета требуется управляющая программа, например Revit, Civil3D, Alias или другая программа, можно выбрать одну или несколько основ в раскрывающемся списке. При поиске пакетов пользователи могут применять фильтры для отображения результатов по основе, что помогает в поиске пакетов, совместимых с предпочтительными программами.

Ключевые слова (необязательно):
Ключевые слова, которые можно использовать в качестве тегов, относящихся к данному пакету, например Generative Design, ML и т. д. В поле поиска можно ввести несколько ключевых слов через пробел, а затем нажать клавишу ENTER.

Группа (необязательно):
Это поле можно использовать для указания группы, к которой принадлежит пакет, например компании или организации.

Веб-сайт (необязательно):
Ссылка на веб-сайт, где могут содержаться дополнительные сведения о пакете, организации или разработчике.

URL-адрес хранилища (необязательно):
Ссылка на репозиторий пакета, где можно просмотреть сведения о разработке пакета или принять участие в ней.

Лицензия (необязательно):
Лицензия, на основании которой был выпущен пакет. Лицензия по умолчанию: MIT.

Правообладатель (необязательно):
Имя лица, которое обладает авторскими правами на пакет. Если оставить это поле пустым, по умолчанию будет указано имя пользователя.

Год авторского права (необязательно):
Год авторского права для пакета. Если оставить это поле пустым, по умолчанию будет указан год публикации пакета.

Добавление файлов пакетов

Содержимое пакета:
В этом разделе, расположенном справа на вкладке «Публикация пакета», можно упорядочить содержимое пакета перед его публикацией.

Добавить файлы/папку:

Выберите и добавьте все файлы, связанные с пакетом. Обычно это папка bin, двоичные файлы и зависимости. Можно добавить отдельные файлы или папки.

После выбора файлов или папок для добавления можно просмотреть, удалить или добавить файлы на странице «Выберите содержимое пакета». На панели слева можно просмотреть структуру папок выбранных файлов, а на панели справа — содержимое каждой папки.

Выберите библиотеки узлов:

Укажите, какие из добавленных в пакет файлов являются библиотеками узлов, установив флажок в столбце «Библиотека узлов». Библиотеки узлов содержат типы и методы, которые Dynamo импортирует как узлы. Обычно в качестве библиотеки узлов можно отметить любую сборку, содержащую типы, которые требуется импортировать в Dynamo. Сборки ZeroTouch, NodeModel или NodeViewCustomization должны быть отмечены как библиотеки узлов.

Для загрузки сборок NodeViewCustomization пользовательского интерфейса они должны быть определены в той же сборке, которая содержит соответствующие определения NodeModel, ИЛИ определены в отдельной сборке. В любом случае сборка должна быть помечена как библиотека узлов. В настоящее время для сборок, помеченных как библиотеки узлов, которые содержат типы NodeModel или NodeCustomization, все другие типы игнорируются. Поэтому важно отделить типы NodeModel/NodeViewCustomization от типов, которые требуется импортировать с помощью импорта ZeroTouch.

Сохранить структуру папок:

Разработчикам пакетов со сложной структурой папок могут доступен параметр «Сохранить структуру папок» под панелью предварительного просмотра содержимого пакета. Этот параметр включен по умолчанию, если пользователь публикует новую версию существующего пакета. В этом случае программа Dynamo не вносит изменения в структуру папок пакета, пакет выгружается как есть.

Включите этот параметр, если нужно настроить пакет определенным образом, чтобы убедиться в его работоспособности и предотвратить возникновение проблем. Если есть сомнения или вы публикуете новый пакет, отключите этот параметр, чтобы разрешить Dynamo настраивать структуру папок.

Проверьте содержимое пакета:
Если содержимое пакета вас устраивает, нажмите кнопку «Далее», чтобы проверить, как будет выглядеть структура файлов пакета после его публикации. Сведения на этой странице доступны только для просмотра. Чтобы внести дополнительные изменения в содержимое пакета, вернитесь на предыдущую страницу. Нажмите кнопку «Готово», чтобы вернуться на страницу сведений о пакете.

Добавьте путь к файлу разметки (необязательно):
В разделе «Путь к файлам разметки» можно указать путь к файлу, который содержит документацию по разметке для пакета (с расширением MD).

Публикация пакета
После заполнения всех необходимых данных пакет можно опубликовать на локальном компьютере или в интернете. При публикации на локальном компьютере пакет публикуется в локальной корневой папке, а при публикации в интернете он выгружается в Package Manager.

Антивирусное сканирование

Для обеспечения безопасной работы пользователей с пакетами Dynamo мы внедрили антивирусное сканирование для новых пакетов. Когда пользователь выгружает пакет, он проверяется на наличие вирусов и становится общедоступным, только если была подтверждена его безопасность. Сканирование выполняется в процессе выгрузки пакета и не влияет на время, через которое пакеты становятся доступны в интернете. Оно по-прежнему составляет 10–15 минут. Если будет обнаружен вирус, на странице поиска Package Manager появится уведомление о том, что выгруженный пакет заражен и не будет опубликован в интернете. Чтобы сделать пакет общедоступным, проверьте файлы, включенные в пакет, и выгрузите новую версию с помощью Package Manager.

Управление пакетами

Список опубликованных пакетов представлен на вкладке «Мои пакеты» в Package Manager, где их можно исключить, опубликовать новую версию и т. д.

Параметры пакета

Параметры пакета, включая новые пути скачивания пакетов и пути поиска пакетов/библиотек, представлены на вкладке «Параметры пакета» в Package Manager. Чтобы изменить параметры, щелкните ссылку «Настройки» на вкладке «Параметры пакета» для перехода к соответствующей панели.
