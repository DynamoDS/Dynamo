Package Manager

O Package Manager do Dynamo é um local único para navegar, instalar e publicar pacotes. Acesse-o na parte superior da navegação em Pacotes > Package Manager.

O que é um pacote?

O Dynamo oferece um grande número de recursos prontos para uso e também mantém uma ampla biblioteca de pacotes que pode estender significativamente a capacidade do Dynamo. Um pacote é uma coleção de nós personalizados ou funcionalidades adicionais. O Package Manager do Dynamo é um portal para a comunidade fazer o download de qualquer pacote que tenha sido publicado on-line. Esses conjuntos de ferramentas foram desenvolvidos por terceiros e pela equipe do Dynamo para estender a funcionalidade central do Dynamo, acessível a todos, e pronta para download com o clique do botão.

Instalar pacotes do Dynamo

É possível procurar e instalar pacotes na guia Procurar pacotes do Package Manager. Use o campo de pesquisa para localizar pacotes específicos. Também é possível aplicar filtros e classificar para restringir a lista para atender aos critérios, como um programa hospedeiro ou falta de dependências.

A compatibilidade de versões é mostrada por meio de ícones em toda a página Pesquisar pacotes:

  • Um sinalizador com marca de seleção verde significa que a versão provavelmente será compatível com sua versão atual do Dynamo e do programa hospedeiro, como o Revit (se estiver usando um).
  • Um sinalizador com um losango vermelho significa que a versão provavelmente será incompatível com sua versão atual do Dynamo e hospedeiro (se estiver usando um).
  • Um sinalizador com um ponto de interrogação azul significa que as informações de compatibilidade não são conhecidas para a versão do pacote. Isso pode ou não funcionar com sua configuração.

Para visualizar mais informações sobre um pacote, clique em Informações ou no nome do pacote. Isso abre um painel lateral no qual é possível ler uma descrição do pacote, encontrar informações da versão e da licença etc.

Para instalar um pacote, clique em Instalar na vista de pesquisa. Se o autor do pacote tiver adicionado informações de compatibilidade ao pacote, a versão compatível mais recente será instalada por padrão. Caso contrário, a versão mais recente será instalada, a menos que você escolha outra versão no menu suspenso. No painel de detalhes do pacote, é possível clicar em Instalar na parte superior de uma versão específica na seção Versões e requisitos do pacote para instalar essa versão.

Para especificar a pasta em que o pacote será instalado, clique na seta do menu suspenso ao lado do botão Instalar.

É possível encontrar uma lista dos pacotes instalados na guia Pacotes instalados do Package Manager. Nessa guia, é possível visualizar os arquivos dos pacotes, desinstalar pacotes e muito mais.

Votar em pacotes

Se desejar expressar seu reconhecimento pelo trabalho de um autor de pacote, será possível votar em um pacote instalado. Vá para Pesquisar informações de pacotes , clique no ícone de coração na parte superior dos detalhes do pacote. Isso só está disponível para pacotes instalados.

Publicação de pacotes do Dynamo

O Package Manager permite publicar pacotes no Dynamo. Abra a guia Publicar um pacote para começar o processo passo a passo de publicação de um pacote. A guia Publicar um pacote solicita todas as informações necessárias e opcionais relacionadas ao pacote que você deseja publicar.

Etapa 1: Detalhes do pacote

Nessa página, é possível adicionar informações sobre o pacote como um todo, bem como a versão individual. Somente o nome, a descrição e o número da versão do pacote são necessários; todos os outros campos são opcionais, mas recomendados para ajudar os usuários a localizar e entender melhor a finalidade do pacote.

Informações do pacote

Nome (obrigatório):

Nome do pacote que você deseja publicar. Esse nome será exibido na biblioteca, na pesquisa de pacotes e no site do Package Manager. Os nomes de pacote devem ter pelo menos 3 caracteres.

Descrição (obrigatório):

A descrição é uma oportunidade para adicionar mais detalhes sobre o pacote, incluindo a listagem de exceções e pré-requisitos. Ela também pode ser usada para instruir os usuários sobre o que o pacote faz e como ele pode ser usado em seus espaços de trabalho.

Para obter informações específicas da versão, é mais eficiente adicionar um link de notas de versão. Observe que as informações de compatibilidade são adicionadas na próxima etapa; portanto, você não precisa adicionar informações de compatibilidade específicas da versão à descrição do pacote.

Grupo:

É possível usar esse campo para especificar um grupo ao qual um pacote pertence, como a empresa/organização.

Palavras-chave:

Palavras-chave que podem ser usadas como identificadores relacionados ao pacote, por exemplo, Projeto generativo, ML etc. As palavras-chave podem ajudar os usuários a encontrar os pacotes por meio da pesquisa. Você pode digitar várias palavras-chave no campo de uma só vez separando-as com uma vírgula e pressionando Enter.

Link do site:

Um link para um site que fornece ao usuário mais informações sobre o pacote ou a organização/autor.

Link do repositório:

Um link para o repositório do pacote que permite que os usuários visualizem ou contribuam para o desenvolvimento do pacote.

Titular dos direitos autorais:

Nome da pessoa que detém os direitos autorais do pacote. Se deixado em branco, o padrão será seu nome de usuário.

Informações da versão

Versão (obrigatório):

O campo de versão está dividido em três seções, principal, secundária e de compilação, o que permite que você, como autor do pacote, carregue novas versões do pacote que seguem a semântica de controle de versão. Os autores de pacotes podem querer publicar mais versões de seus pacotes com recursos novos e aprimorados, aperfeiçoar alguns detalhes que não foram considerados na primeira versão ou corrigir uma falha.

Link das notas de versão:

Um link para as notas de versão desta versão. É possível adicionar um link separado para cada versão publicada.

Ano dos direitos autorais:

O ano dos direitos autorais do pacote. Se deixado em branco, o padrão será o ano em que o pacote foi publicado.

Licença:

Uma licença com base na qual o pacote é liberado. A licença se aplica à última versão do pacote. Se deixado em branco, o pacote utilizará a licença MIT.

Etapa 2: Compatibilidade

Adicione informações de compatibilidade à versão do pacote. Essas informações ajudarão os usuários a localizar e instalar apenas as versões do pacote que serão compatíveis com sua configuração, reduzindo possíveis problemas e evitando confusão. Quando as informações de compatibilidade de versão são atribuídas, os usuários podem filtrar somente por versões compatíveis e são notificados sobre o status de compatibilidade por meio dos ícones na guia Procurar pacotes.

Para começar a adicionar informações de compatibilidade, selecione se a versão do pacote funciona somente em um programa hospedeiro, como o Revit ou o Civil 3D; em um programa hospedeiro e no Dynamo Core; ou somente no Dynamo Core. Após selecionar, você verá controles de interface do usuário adicionais para começar a atribuir uma ou mais versões ao pacote.

Para o Dynamo Core ou um programa hospedeiro, é possível usar os menus suspensos De e Até para selecionar um intervalo ou adicionar versões uma a uma no menu suspenso Adicionar versões individuais. No menu suspenso Para, também há a opção de selecionar um caractere curinga, por exemplo, 2025.x (todas as versões). Isso significa que a versão do pacote é compatível com todas as versões dessa versão principal.

À medida que adiciona versões, é possível adicioná-las à tabela Todas as versões compatíveis, à direita, clicando no botão Adicionar no meio. Se você cometer um erro, será possível remover versões dessa tabela clicando no ícone x no identificador de versão.

Os autores de pacotes podem atualizar as informações de compatibilidade de seus pacotes do Dynamo no site do Dynamo Package Manager (www.dynamopackages.com). Para fazer isso, faça login no site, navegue até a guia Meus pacotes, localize seu pacote e siga as etapas descritas acima.

Passo 3: Selecionar arquivos

Adicionar arquivos/diretório:

Selecione e adicione todos os arquivos relacionados ao pacote. Geralmente, a pasta bin, os binários e as dependências são adicionados. É possível adicionar arquivos ou pastas individuais.

Selecionar arquivos do pacote:

Essa seção permite organizar o conteúdo do pacote antes da publicação. Após selecionar os arquivos ou as pastas a serem adicionados, será possível visualizar, remover ou adicionar arquivos na página Selecionar arquivos do pacote. É possível visualizar a estrutura de pastas dos arquivos selecionados no painel esquerdo e o conteúdo de cada pasta no painel direito.

Para remover um arquivo do pacote, clique nele e, em seguida, clique no ícone da lixeira (remover) na linha superior. Também é possível selecionar e remover vários arquivos dessa maneira. Para remover pastas inteiras do pacote, clique na pasta e, em seguida, clique no ícone da lixeira que é exibido ao lado dela.

Selecionar bibliotecas de nós:

Especifique quais dos arquivos adicionados ao pacote são as bibliotecas de nós marcando a caixa na coluna Biblioteca de nós. As bibliotecas de nós contêm os tipos e os métodos que o Dynamo importará como nós. Em geral, você deve marcar qualquer montagem como uma biblioteca de nós que contenha tipos que deseja importar para o Dynamo. As montagens ZeroTouch, NodeModel ou NodeViewCustomization devem ser marcadas como uma biblioteca de nós.

Para carregar as montagens NodeViewCustomizations da interface do usuário, elas devem ser definidas na mesma montagem que contém as definições correspondentes do NodeModel OU definidas em uma montagem separada. Em ambos os casos, a montagem deve ser identificada como uma biblioteca de nós. No momento, no caso das montagens marcadas como biblioteca de nós que contêm os tipos NodeModel ou NodeCustomization, todos os outros tipos serão ignorados. Por esse motivo, é importante separar os tipos NodeModel/NodeViewCustomization dos tipos que você deseja importar usando a importação ZeroTouch.

Manter estrutura de pastas:

Ao criar um pacote, o Dynamo reorganiza automaticamente os arquivos do pacote em pastas bin, doc, dyf e extras. Os autores de pacotes avançados com estruturas de pastas de pacotes complexas podem optar por marcar a caixa de seleção Manter estrutura de pastas abaixo do painel de visualização do conteúdo do pacote. Quando essa opção estiver ativada, o Dynamo não fará alterações na estrutura de pastas do pacote; em vez disso, os arquivos do pacote serão carregados como estão.

Os pacotes precisam ser configurados de uma determinada maneira para garantir que funcionem e para evitar problemas. Quando estiver em dúvida, verifique se a caixa de seleção está desmarcada para permitir que o Dynamo configure a estrutura de pastas.

Passo 4: Visualizar arquivos

Quando estiver satisfeito com o conteúdo do pacote, clique em Avançar para ter uma visualização de como os arquivos do pacote serão configurados depois que forem publicados. Essa página é somente visualização; para fazer alterações adicionais no conteúdo do pacote, volte para a etapa anterior.

Etapa 5: Confirmar

Essa etapa dá a você a chance de ver todos os detalhes do pacote, compatibilidade e arquivos antes da publicação. Para evitar alterações acidentais, as opções selecionadas nas etapas anteriores são somente leitura na página Confirmar. Para alterá-las, volte para as etapas anteriores usando o botão Voltar ou os controles de navegação na parte superior da página.

Adicionar caminho do arquivo markdown (opcional):

Você tem a opção de especificar um caminho de arquivo que hospeda o documentação do markdown (.md) para o pacote na seção Caminho do arquivo markdown. A documentação do nó será exibida no Navegador de documentação, disponível em Extensões, no menu de navegação superior do Dynamo, ou clicando com o botão direito do mouse em um nó e selecionando Ajuda.

Dependências do pacote externo:

Com base nos arquivos selecionados na Etapa 3, o Dynamo detecta as dependências do pacote externo, que estão listadas aqui.

Publicar o pacote

Após preencher todos os detalhes necessários, será possível publicar o pacote localmente ou on-line. Ao publicar localmente, o pacote será publicado no diretório raiz local. Ao publicar on-line, o pacote será carregado no Package Manager para que seja pesquisável por outros usuários depois de passar na verificação antivírus. Ele também está localizado na guia Meus pacotes.

Verificações antivírus

Para fornecer um ambiente seguro a todos os usuários de pacotes do Dynamo, introduzimos verificações antivírus para novos pacotes. Quando um usuário carregar um pacote, ele será verificado para detectar se existem vírus e somente ficará disponível publicamente depois que for marcado como seguro. A verificação ocorrerá durante o processo de carregamento do pacote e não levará mais tempo do que a duração usual de alguns minutos para que o novo pacote esteja disponível on-line.

Se o pacote estiver marcado como infectado, a página de pesquisa do Package Manager exibirá uma notificação informando que o pacote carregado foi considerado infectado e não será publicado on-line. Para tornar o pacote disponível publicamente, verifique os arquivos incluídos no pacote e carregue uma nova versão através do Package Manager.

Gerenciamento dos pacotes

É possível encontrar uma lista dos pacotes publicados na guia Meus pacotes do Package Manager, na qual é possível tornar pacotes obsoletos, publicar uma nova versão etc.

Tornar um pacote obsoleto

Como proprietário de um pacote, você pode tornar um pacote obsoleto. Isso coloca uma etiqueta no pacote para notificar outros usuários de que o pacote foi descontinuado e não é mais mantido. A pesquisa de pacotes por padrão filtra pacotes obsoletos, embora você possa aplicar o filtro Obsoleto para incluí-los nos resultados da pesquisa.

Para cancelar a obsolescência de um pacote, localize o pacote no Package Manager na guia Pacotes instalados ou Meus pacotes. Clique no menu Opções (três pontos) e selecione Obsoleto.

Transferir a propriedade de um pacote

No momento, não é possível transferir a propriedade do pacote por meio do Package Manager. Você pode solicitar que a equipe do Dynamo adicione mais um proprietário. Observe que não podemos remover proprietários existentes, apenas adicionar mais responsáveis pela manutenção do pacote.

Se você desejar adicionar uma conta como proprietária ao pacote existente, envie um e-mail para dynamoteam@dynamobim.org. Certifique-se de fornecer o nome do pacote e o nome da conta que deseja adicionar.

Excluir um pacote

Se você desejar enviar uma solicitação de exclusão, envie um e-mail para dynamoteam@dynamobim.org com a linha de assunto “Solicitação de exclusão de pacotes” e certifique-se de fornecer o nome do pacote. Assim que a solicitação for recebida, um e-mail de confirmação será enviado para o e-mail do autor associado ao pacote. Depois de aprovada a solicitação, a exclusão ocorrerá no prazo de 30 dias. Se o autor não responder no prazo de 30 dias, será necessário enviar uma nova solicitação de exclusão.

Observe que a exclusão de um pacote não libera o nome do pacote para reutilização por novos pacotes.

Configurações do pacote

É possível visualizar as configurações relacionadas ao pacote na guia Configurações do pacote do Package Manager, incluindo novos caminhos de download de pacotes e caminhos de pesquisa de pacote/biblioteca. Para editar as configurações, clique no link Preferências na guia Configurações do pacote para acessar o painel Preferências.
