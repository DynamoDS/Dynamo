Package Manager

Dynamo Package Manager는 패키지를 검색, 설치 및 게시하기 위한 원스톱 솔루션입니다. 패키지 Package Manager의 상단 탐색에서 액세스할 수 있습니다.

패키지란 무엇입니까?

Dynamo는 다양한 기능을 기본적으로 제공하고, Dynamo의 기능을 크게 확장할 수 있는 광범위한 패키지 라이브러리도 유지합니다. 패키지는 사용자 노드 또는 추가 기능의 모음입니다.Dynamo Package Manager는 커뮤니티에서 온라인으로 게시된 패키지를 다운로드할 수 있는 포털입니다.이러한 도구 세트는 Dynamo의 핵심 기능을 확장하고, 누구나 액세스할 수 있으며, 버튼 클릭으로 즉시 다운로드할 수 있도록 타사와 Dynamo 팀이 함께 개발했습니다.

Dynamo 패키지 설치

Package Manager의 패키지 검색 탭에서 패키지를 찾아 설치할 수 있습니다. 검색 필드를 사용하여 특정 패키지를 찾습니다. 또한 필터 및 정렬을 적용하여 호스트 프로그램 또는 종속성 부족과 같은 기준에 맞게 리스트 범위를 좁힐 수 있습니다.

버전 호환성은 패키지 검색 페이지 전체에서 아이콘을 통해 표시됩니다.

  • 녹색 확인 표시가 있는 플래그는 해당 버전이 현재 버전의 Dynamo 및 호스트 프로그램(예: Revit(사용하는 경우))과 호환될 가능성이 높음을 의미합니다.
  • 빨간색 다이아몬드가 있는 플래그는 해당 버전이 현재 버전의 Dynamo 및 호스트(사용하는 경우)와 호환되지 않을 수 있음을 의미합니다.
  • 파란색 물음표가 있는 플래그는 패키지 버전에 대한 호환성 정보를 알 수 없음을 의미합니다. 설정에서 작동하거나 작동하지 않을 수도 있습니다.

패키지에 대한 자세한 내용을 보려면 정보 또는 패키지 이름을 클릭합니다. 그러면 패키지에 대한 설명을 읽고, 버전 및 라이센스 정보 등을 찾을 수 있는 측면 패널이 열립니다.

패키지를 설치하려면 검색 뷰에서 설치를 클릭합니다. 패키지 작성자가 패키지에 호환성 정보를 추가한 경우 기본적으로 호환되는 최신 버전이 설치됩니다. 그렇지 않으면 드롭다운 메뉴에서 다른 버전을 선택하지 않는 한 최신 버전이 설치됩니다. 패키지 상세정보 패널의 버전 및 패키지 요구 사항 섹션에서 특정 버전 상단에 있는 설치를 클릭하여 해당 버전을 설치할 수 있습니다.

패키지를 설치할 폴더를 지정하려면 설치 버튼 옆에 있는 드롭다운 화살표를 클릭합니다.

Package Manager의 설치된 패키지 탭에서 설치한 패키지 리스트를 찾을 수 있습니다. 이 탭에서 패키지 파일을 보고 패키지를 설치제거하는 등의 작업을 할 수 있습니다.

패키지에 대한 투표

패키지 작성자의 작업에 감사의 뜻을 표하고 싶다면 설치한 패키지에 투표할 수 있습니다. 패키지 정보 검색으로 이동하여 패키지 상세정보 상단에 있는 하트 아이콘을 클릭합니다. 설치된 패키지에만 사용할 수 있습니다.

Dynamo 패키지 게시

Package Manager를 사용하여 Dynamo 내에 패키지를 게시할 수 있습니다. 패키지 게시 탭을 열어 패키지를 게시하는 단계별 프로세스를 시작합니다. 패키지 게시 탭에서 게시하려는 패키지와 관련된 모든 필수 정보 및 선택적 정보를 묻습니다.

1단계: 패키지 상세정보

이 페이지에서 전체 패키지와 개별 버전에 대한 정보를 추가할 수 있습니다. 패키지 이름, 설명 및 버전 번호만 필요합니다. 다른 모든 필드는 선택 사항이지만 사용자가 패키지를 찾고 그 용도를 더 잘 이해하는데 도움이 되므로 권장됩니다.

패키지 정보

이름(필수):

게시할 패키지의 이름입니다. 이는 라이브러리, 패키지 검색 및 Package Manager 웹사이트에 표시됩니다. 패키지 이름의 길이는 3자 이상이어야 합니다.

설명(필수):

설명에는 예외 및 전제 조건 리스트를 포함하여 패키지에 대한 상세정보를 추가할 수 있습니다. 또한 패키지의 기능 및 작업 영역에서 패키지를 사용할 수 있는 방법에 대해 사용자를 교육하는 데에도 사용할 수 있습니다.

버전별 정보를 보려면 릴리즈 정보 링크를 추가하는 것이 더 효율적입니다. 호환성 정보는 다음 단계에서 추가되므로 패키지 설명에 버전별 호환성 정보를 추가할 필요가 없습니다.

그룹:

이 필드는 패키지가 속한 그룹(예: 회사/조직)을 지정하는 데 사용할 수 있습니다.

키워드:

패키지와 관련된 태그로 사용할 수 있는 키워드(예: 제너레이티브 디자인, ML 등)입니다. 키워드는 사용자가 검색을 통해 패키지를 찾는 데 도움이 될 수 있습니다. 여러 키워드를 쉼표로 구분한 다음 Enter 키를 눌러 필드에 한 번에 입력할 수 있습니다.

웹사이트 링크:

패키지 또는 조직/작성자에 대한 자세한 정보를 사용자에게 제공하는 웹사이트의 링크입니다.

리포지토리 링크:

사용자가 패키지 개발을 확인하거나 패키지 개발에 기여할 수 있는 패키지 리포지토리의 링크입니다.

저작권 소유자:

패키지의 저작권을 보유한 사람의 이름입니다. 비워 두면 사용자 이름이 기본값으로 설정됩니다.

버전 정보

버전(필수):

버전 필드는 세 개의 섹션인 주요, 보조 및 빌드로 나뉘며, 이를 통해 패키지 작성자가 버전 관리 의미 체계를 따르는 패키지의 새 버전을 업로드할 수 있습니다. 패키지 작성자는 새로운 기능과 향상된 기능이 포함된 패키지의 추가 버전을 게시하고 첫 번째 버전에서 다루지 못한 부분을 개선하거나 버그를 해결할 수 있습니다.

릴리즈 노트 링크:

해당 버전의 릴리즈 정보에 대한 링크입니다. 게시하는 각 버전에 별도의 링크를 추가할 수 있습니다.

저작권 연도:

패키지의 저작권 연도입니다. 비워 두면 패키지가 게시된 연도가 기본값으로 설정됩니다.

라이센스:

패키지가 릴리즈되는 라이센스입니다. 라이센스는 최신 패키지 버전에 적용됩니다. 비워 두면 MIT에 따라 패키지에 라이선스가 부여됩니다.

2단계: 호환성

패키지 버전에 대한 호환성 정보를 추가합니다. 이 정보는 사용자가 자신의 설정과 호환되는 패키지 버전만 찾아 설치하는 데 도움이 되므로 잠재적인 문제와 혼동을 줄일 수 있습니다. 버전 호환성 정보가 지정되면 사용자는 호환되는 버전으로만 필터링할 수 있으며 패키지 검색 탭의 아이콘을 통해 호환성 상태를 확인할 수 있습니다.

호환성 정보를 추가하려면 패키지 버전이 호스트 프로그램(예: Revit 또는 Civil 3D)에서만 작동하는지, 호스트 프로그램 및 Dynamo Core에서 작동하는지, 또는 Dynamo Core에서만 작동하는지 선택합니다. 선택하면 하나 이상의 버전을 패키지에 할당할 수 있는 추가 UI 컨트롤이 표시됩니다.

Dynamo Core 또는 호스트 프로그램의 경우 시작 및 끝 드롭다운을 사용하여 범위를 선택하거나 개별 버전 추가 드롭다운에서 버전을 하나씩 추가할 수 있습니다. 끝 드롭다운에서 와일드카드를 선택할 수도 있습니다(예: 2025.x(모든 버전)). 이는 패키지 버전이 해당 주 릴리즈 내의 모든 버전과 호환됨을 의미합니다.

버전을 추가할 때 중간에 있는 추가 버튼을 클릭하여 오른쪽의 모든 호환 버전 표에 추가할 수 있습니다. 잘못 추가한 경우 버전 태그에서 x 아이콘을 클릭하여 이 표에서 버전을 제거할 수 있습니다.

패키지 작성자는 Dynamo Package Manager 웹사이트(www.dynamopackages.com)를 통해 Dynamo 패키지의 호환성 정보를 업데이트할 수 있습니다. 이렇게 하려면 웹사이트에 로그인하고 내 패키지 탭으로 이동하여 패키지를 찾은 다음 위에 설명된 단계를 따릅니다.

3단계: 파일 선택

파일/디렉토리 추가:

패키지와 관련된 모든 파일을 선택하여 추가합니다.일반적으로 bin 폴더, 바이너리 및 종속성이 추가됩니다.개별 파일 또는 폴더를 추가할 수 있습니다.

패키지 파일 선택:

이 섹션에서는 게시하기 전에 패키지 컨텐츠를 구성할 수 있습니다. 추가할 파일 또는 폴더를 선택한 후 패키지 파일 선택 페이지에서 파일을 보거나 제거하거나 추가할 수 있습니다. 왼쪽 패널에서 선택한 파일의 폴더 구조를 보고 오른쪽 패널에서 각 폴더의 컨텐츠를 볼 수 있습니다.

패키지에서 파일을 제거하려면 해당 파일을 클릭한 다음 상단 행의 휴지통(제거) 아이콘을 클릭합니다. 이 방법으로 여러 파일을 선택하고 제거할 수도 있습니다. 패키지에서 전체 폴더를 제거하려면 폴더를 클릭한 다음 옆에 나타나는 휴지통 아이콘을 클릭합니다.

노드 라이브러리 선택:

노드 라이브러리 열의 상자를 선택하여 패키지에 추가한 파일 중 노드 라이브러리에 해당하는 파일을 지정합니다. 노드 라이브러리에는 Dynamo에서 노드로 가져올 유형 및 메서드가 포함되어 있습니다. 일반적으로 모든 조합은 Dynamo로 가져올 유형이 포함된 노드 라이브러리로 표시해야 합니다. ZeroTouch, NodeModel 또는 NodeViewCustomization 조합은 노드 라이브러리로 표시해야 합니다.

UI NodeViewCustomizations를 로드하려면 해당 NodeModel 정의를 포함하는 동일한 조함에 정의되거나 별도의 조합에 정의되어야 합니다. 어느 경우든 조합에는 노드 라이브러리라는 태그를 지정해야 합니다. 현재 NodeModel 또는 NodeCustomization 유형을 포함하는 노드 라이브러리로 표시된 조합의 경우 다른 모든 유형은 무시됩니다. 이 때문에 ZeroTouch 가져오기를 사용하여 가져올 유형에서 NodeModel/NodeViewCustomization 유형을 분리하는 것이 중요합니다.

폴더 구조 유지:

패키지를 작성할 때 Dynamo는 패키지 파일을 bin, doc, dyf 및 extra 폴더로 자동으로 재구성합니다. 패키지 폴더 구조가 복잡한 고급 패키지 작성자는 패키지 컨텐츠 미리보기 패널 아래에 있는 폴더 구조 유지 확인란을 선택할 수 있습니다. 이 옵션을 활성화하면 Dynamo가 패키지 폴더 구조를 변경하지 않으며, 패키지 파일이 그대로 업로드됩니다.

패키지가 제대로 작동하고 문제가 발생하지 않도록 패키지를 특정 방식으로 설정해야 합니다. 확실하지 않은 경우 Dynamo에서 폴더 구조를 설정할 수 있도록 해당 확인란을 선택 취소해야 합니다.

4단계: 파일 미리보기

패키지 컨텐츠가 만족스러우면 다음을 클릭하여 패키지 파일이 게시된 후 설정되는 방식에 대한 미리보기를 확인합니다. 이 페이지는 보기 전용입니다. 패키지 컨텐츠를 추가로 변경하려면 이전 단계로 돌아갑니다.

5단계: 확인

이 단계에서는 게시하기 전에 모든 패키지 상세정보, 호환성 및 파일을 볼 수 있습니다. 실수로 변경되지 않도록 이전 단계에서 선택한 옵션은 확인 페이지에서 읽기 전용으로 표시됩니다. 변경하려면 뒤로 버튼 또는 페이지 상단에 있는 탐색 컨트롤을 사용하여 이전 단계로 돌아갑니다.

Markdown 파일 경로 추가(선택 사항):

Markdown 파일 경로 섹션에서 패키지에 대한 Markdown(.md) 문서를 호스팅하는 파일 경로를 지정할 수 있습니다. 노드 문서는 문서 탐색기에 표시되며, 이 탐색기는 Dynamo의 상단 탐색 메뉴의 확장 아래에 표시되거나 노드를 마우스 오른쪽 버튼으로 클릭하고 도움말을 선택하면 사용할 수 있습니다.

외부 패키지 종속성:

3단계에서 선택한 파일을 기반으로 Dynamo는 여기에 나열된 외부 패키지 종속성을 탐지합니다.

패키지 게시

필요한 모든 상세정보를 입력한 후 패키지를 로컬 또는 온라인으로 게시할 수 있습니다. 로컬로 게시할 때 패키지는 로컬 루트 디렉토리에 게시됩니다. 온라인으로 게시할 때 패키지는 바이러스 백신 검사를 통과한 후 다른 사용자가 검색할 수 있도록 Package Manager에 업로드됩니다. 내 패키지 탭에서도 찾을 수 있습니다.

바이러스 백신 스캔

모든 Dynamo 패키지 사용자에게 안전한 환경을 제공하기 위해 새 패키지에 대한 바이러스 백신 스캔을 도입했습니다. 사용자가 패키지를 업로드하면 바이러스 스캔이 수행되고 안전 플래그가 지정된 후에만 패키지를 공개적으로 사용할 수 있습니다. 스캔은 패키지 업로드 프로세스 중에 수행되며 새 패키지를 온라인으로 사용할 수 있도록 일반적으로 몇 분 동안에는 추가되지 않습니다.

패키지가 감염된 것으로 플래그가 지정되면 업로드된 패키지가 감염된 것으로 확인되었고 온라인으로 게시되지 않는다는 알림이 Package Manager 검색 페이지에 표시됩니다. 패키지를 공개적으로 사용할 수 있도록 하려면 패키지에 포함된 파일을 확인하고 Package Manager를 통해 새 버전을 업로드하십시오.

패키지 관리

게시된 패키지의 리스트는 Package Manager의 내 패키지 탭에서 찾을 수 있고, 여기에서 패키지를 사용 중단하고 새 버전을 게시하는 등의 작업을 수행할 수 있습니다.

패키지 사용 중단

패키지 소유자는 패키지를 사용 중단할 수 있습니다. 이렇게 하면 패키지에 태그가 추가되어 다른 사용자에게 해당 패키지가 중단되었으며 더 이상 유지 관리되지 않음을 알립니다. 패키지 검색은 기본적으로 사용되지 않는 패키지를 필터링하여 제외하지만 사용되지 않음 필터를 적용하여 검색 결과에 포함할 수 있습니다.

패키지를 사용 중단하려면 Package Manager의 설치된 패키지 또는 내 패키지 탭에서 패키지를 찾습니다. 옵션 메뉴(세 점)를 클릭하고 사용 중단을 선택합니다.

패키지 소유권 양도

현재는 Package Manager를 통해 패키지 소유권을 양도할 수 없습니다. Dynamo 팀에 다른 소유자를 추가하도록 요청할 수 있습니다. 기존 소유자는 제거할 수 없으며 패키지의 유지 관리자만 더 추가할 수 있습니다.

기존 패키지에 계정을 소유자로 추가하려면 dynamoteam@dynamobim.org로 이메일을 보내주십시오. 추가하려는 패키지 이름과 계정 이름을 입력해야 합니다.

패키지 삭제

삭제 요청을 제출하려면 '패키지 삭제 요청'이라는 제목으로 이메일을 dynamoteam@dynamobim.org로 보내고 패키지 이름을 입력해야 합니다. 요청이 수신되면 패키지와 연결된 패키지 작성자의 이메일로 확인 이메일이 전송됩니다. 작성자가 요청을 승인하면 30일 이내에 삭제됩니다. 작성자가 30일 이내에 응답하지 않으면 새 삭제 요청이 필요합니다.

패키지를 삭제해도 새 패키지에서 다시 사용할 수 있는 것은 아닙니다.

패키지 설정

Package Manager의 패키지 설정 탭에서 새 패키지 다운로드 경로 및 패키지/라이브러리 검색 경로를 포함하여 패키지 관련 설정을 볼 수 있습니다. 설정을 편집하려면 패키지 설정 탭에서 기본 설정 링크를 클릭하여 기본 설정 패널에 액세스합니다.
