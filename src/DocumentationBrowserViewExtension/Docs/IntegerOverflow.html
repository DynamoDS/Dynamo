The operation resulted in an integer overflow

Integer overflow occurs when a numeric value falls outside of the range in which that integer can be represented - either higher than the maximum, or lower than the minimum. This may cause the integer value to wrap around the maximum or minimum value and generate an unexpected result.

How to fix

When working with very large numbers, try to use the double (decimal placed number type) instead of the int (whole number type). Both are used to represent numbers but differ greatly in the range of values they support:

Type Minimum (approximate) Maximum (approximate)
int -9.2e18 9.2e18
double -1.7e308 1.7e308

In the table above, the number after e represents how many times you should multiply by 10 to get the actual minimum or maximum. As you can see, the difference is of around 290 digits, which makes it a huge number and hard to reach in practice.
