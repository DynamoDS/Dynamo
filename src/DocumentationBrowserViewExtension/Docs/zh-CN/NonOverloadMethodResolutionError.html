期望参数类型，但调用其他参数类型

此错误消息指示您使用的节点需要特定类型的输入，但收到的某类输入不能使用。 它通常后跟附加信息，该信息涉及哪些类型已传递到使用的节点以及节点需要接收的类型。 触发此错误的一些最常见操作包括:

  • Element.SetParameterByName - 尝试使用 Element.Parameters 节点来检查图元中的参数。 阅读更多 Dynamobim 论坛上讨论“expects argument type setparameter”的帖子。
  • Geometry.BoundingBox - 尝试改用 Element.BoundingBox，如此处所述。 阅读更多 Dynamobim 论坛上讨论“expects argument type geometry”的帖子。

如何修复

上述部分提到了一些可以用于从此错误中恢复的方法。 通常，检查使用的节点期望的输入类型。 如果可能，将传递到节点的输入更改为其期望类型，或检查以查看其他节点是否可用于获取所需的功能。 请注意，可以使用 Object.Type 节点检查输入类型。
